# Референсний Чек-лист WCAG 2.2 Level AA

## 1. Сприйнятливість (Perceivable)

| Критерій  | Назва                  | Короткий опис                                                                         |
| :-------- | :--------------------- | :------------------------------------------------------------------------------------ |
| **1.1.1** | Non-text Content       | Увесь нетекстовий контент має текстові альтернативи (наприклад, `alt` для `<img>`).   |
| **1.3.1** | Info and Relationships | Структура сторінки (заголовки, списки, таблиці, форми) має бути семантично коректною. |
| **1.4.1** | Use of Color           | Колір не є єдиним засобом передачі інформації.                                        |
| **1.4.3** | Contrast (Minimum)     | Візуальний контраст тексту до фону мінімум **4.5:1**.                                 |

## 2. Керованість (Operable)

| Критерій   | Назва                     | Короткий опис                                                                                                       |
| :--------- | :------------------------ | :------------------------------------------------------------------------------------------------------------------ |
| **2.1.1**  | Keyboard                  | Весь функціонал доступний за допомогою клавіатури без "пастки фокусу".                                              |
| **2.4.7**  | Focus Visible             | Індикатор фокусу елемента (зазвичай рамка) чітко видимий.                                                           |
| **2.4.13** | **Focus Not Obscured**    | Елемент у фокусі не повинен бути повністю прихований іншими елементами (наприклад, sticky-хедером).                 |
| **2.5.3**  | Label in Name             | Візуальний підпис кнопки/поля відповідає її програмній назві (для скрінрідерів).                                    |
| **2.5.7**  | **Dragging Movements**    | Якщо дія вимагає перетягування (Drag-and-Drop), має бути альтернатива в один клік (наприклад, кнопки "Вгору/Вниз"). |
| **2.5.8**  | **Target Size (Minimum)** | Клікабельні області повинні мати мінімальний розмір **24x24 CSS пікселів**.                                         |

## 3. Зрозумілість (Understandable)

| Критерій  | Назва                                     | Короткий опис                                                                                          |
| :-------- | :---------------------------------------- | :----------------------------------------------------------------------------------------------------- |
| **3.2.3** | Consistent Navigation                     | Навігаційні механізми (меню, посилання) повторюються на всіх сторінках.                                |
| **3.3.2** | Labels or Instructions                    | Усі поля форми мають коректні, пов'язані підписи (`<label for="...">`).                                |
| **3.3.4** | Error Prevention (Legal, Financial, Data) | Для критичних форм (транзакції, персональні дані) є механізми перевірки, підтвердження або скасування. |

## 4. Надійність (Robust)

| Критерій  | Назва             | Короткий опис                                                         |
| :-------- | :---------------- | :-------------------------------------------------------------------- |
| **4.1.1** | Parsing           | HTML-код має бути валідним і не містити серйозних помилок у розмітці. |
| **4.1.2** | Name, Role, Value | Усі UI-компоненти мають програмно визначені ім'я, роль та стан.       |
