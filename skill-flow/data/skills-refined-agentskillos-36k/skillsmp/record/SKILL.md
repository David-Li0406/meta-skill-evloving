# Record

会話で生じた情報を、人間や別セッションのLLMが後から文脈を復元できる粒度で記録する。

## 品質基準: 復元可能とは何か

記録を読んだ人が以下を再構築できること：

- 何が論点だったか
- どんな選択肢が検討されたか
- なぜその選択肢が採用/不採用になったか
- 最終的に何がどういう根拠で決まったか

逆に言えば、結論だけでは不十分。結論に至った推論過程を含めて初めて復元可能になる。

## 焦点の選択

記録の状況に応じて、適切な焦点を選択する：

| 焦点 | 適する状況 | 構造の特徴 |
|------|-----------|-----------|
| 意思決定 | 技術選定、設計判断、方針決定 | 選択肢の比較表 + 不採用理由 + 決定根拠 |
| 時系列 | 調査過程、デバッグ経緯、試行錯誤 | 時間順の発見・判明事項 + 各段階での仮説変遷 |
| 文脈復元 | セッション引き継ぎ、今後の参照用 | 前提知識 + 現状 + 未決事項 + 次のアクション |

複数の焦点が混在する場合はそのまま混在させて良い。無理に1つに絞らない。

## 構造のガイドライン

1. サマリ（1-2行で核心を述べる）
2. 背景・動機（なぜこれが必要だったか）
3. 検討の経緯（焦点に応じた本体部分）
4. 結論・現状（最終的にどうなったか）

各セクション内では、比較表や箇条書きを活用して情報密度を高める。

## 出力先の判断

- ファイル: 長期保存が必要な場合、他のセッションから参照する場合
- PR本文: コード変更に紐づく意思決定の記録
- 会話内: 即座に確認したい場合、ファイル化するほどでもない場合

ユーザーが指定しない場合は確認する。
