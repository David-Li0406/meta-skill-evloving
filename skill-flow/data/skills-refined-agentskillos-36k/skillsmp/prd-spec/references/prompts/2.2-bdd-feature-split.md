<task>
Split `docs/prd/scenarios.md` into one Gherkin `.feature` file per User Story (sections like "### US-XXX: Title"). Preserve all scenario text and IDs exactly. Save under `docs/prd/features/`. Then generate `docs/prd/features-manifest.json` to track all scenarios.
</task>

<rules>
- Keep scenario content verbatim; keep "Scenario ID" comments intact.
- Feature name: use the story title after "US-XXX:". If a "Feature:" line exists beneath the story, use that.
- Filename: `us_XXX_<snake_case_title>.feature` (e.g., `us_001_problem_and_audience_definition.feature`).
- Priority tags: infer from section (P0/P1/P2) → add `@priority-p0` etc.
- Background: only if multiple scenarios share clear common Given(s).
- Cross-cutting sections (Analytics, Security, Integration, Performance): place in `features/cross_cutting/` (e.g., `analytics.feature`), preserving IDs.
- Ambiguity: add a top file comment `# TODO: review` and proceed.
- Add `@meta` JSON block to each `.feature` file for scenario metadata (see format below).
</rules>

<meta_block_format>
Each `.feature` file should include a `@meta` block as a comment at the top, after any tags:

```gherkin
# @meta
# {
#   "story_id": "US-004",
#   "title": "Multi-Platform Partner Search",
#   "priority": "P0",
#   "success_metrics": ["100+ partners discovered within 10 minutes"],
#   "business_rationale": "Users need to discover partners across multiple platforms efficiently"
# }
```
</meta_block_format>

<manifest_generation>
After creating all `.feature` files, generate `docs/prd/features-manifest.json`:

```json
{
  "productName": "Product name from PRD",
  "generatedAt": "ISO-8601 timestamp",
  "lastUpdated": "ISO-8601 timestamp",
  "summary": {
    "total": 15,
    "passing": 0,
    "failing": 15,
    "byPriority": { "P0": 6, "P1": 5, "P2": 4 },
    "passingByPriority": { "P0": 0, "P1": 0, "P2": 0 }
  },
  "features": [
    {
      "scenarioId": "US-004-S01",
      "userStoryId": "US-004",
      "featureFile": "us_004_multi_platform_partner_search.feature",
      "scenarioTitle": "System discovers 100+ partners across all platforms within target time",
      "priority": "P0",
      "category": "functional",
      "description": "From @meta.business_rationale or scenario context",
      "steps": ["Step 1 derived from Given/When/Then", "Step 2..."],
      "successMetrics": ["From @meta.success_metrics if present"],
      "verificationMethod": "browser|automated|manual",
      "passes": false,
      "lastVerified": null,
      "testFile": null,
      "notes": ""
    }
  ]
}
```

**Step derivation rules:**
- `Given` → Setup action (imperative: "Select", "Configure", "Create")
- `When` → Trigger action (imperative: "Initiate", "Click", "Submit")
- `Then/And` → Verification ("Verify", "Check", "Confirm")

**Verification method inference from scenario content:**
| Keywords/Patterns | Method |
|-------------------|--------|
| "real-time", "counter shows", "user sees", "UI displays", "user clicks" | `browser` |
| "user closes browser", "user returns", "manual input", "visual", "layout" | `browser` |
| "API returns", "retries with", "gracefully handles", "database" | `automated` |
| "results include", "distribution shows", "tagged with", "logs show" | `automated` |
| Visual design, layout, styling, subjective quality | `manual` |

Default to `automated` if uncertain.
</manifest_generation>
