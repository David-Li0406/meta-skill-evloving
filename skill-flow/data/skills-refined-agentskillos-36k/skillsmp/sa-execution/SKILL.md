# 执行写作计划

## 概述

加载计划，批判性地审阅，分批执行任务，并在批次之间报告以供审阅。

**核心原则：** 带检查点的批量执行，供主笔（架构师）审阅。

**开始时声明：** “我正在使用 execution 技能来实施此计划。”

## 流程

### 步骤 1: 加载和审阅计划
1. 读取计划文件
2. 批判性审阅 - 识别关于计划的任何问题或疑虑
3. 如果有疑虑：在开始之前向你的人类伙伴提出
4. 如果无疑虑：创建 TodoWrite 并继续

### 步骤 2: 执行批次
**默认：前 3 个任务**

对于每个任务：
1. 标记为 in_progress
2. 严格遵循每个步骤（计划有详细的步骤）
3. 运行指定的验证
4. 标记为 completed

### 步骤 3: 报告
当批次完成时：
1. 展示已撰写的内容
2. 展示验证输出（逻辑检查结果）
3. 说：“准备好接受反馈。”

### 步骤 4: 继续
根据反馈：
1. 如果需要，应用更改
2. 执行下一个批次
3. 重复直到完成

### 步骤 5: 完成草稿开发

所有任务完成并验证后：
- 宣布：“我正在使用 finalization 技能来完成这项工作。”
- **REQUIRED SUB-SKILL:** Use superpowers:sa-finalization
- 遵循该技能来验证论证逻辑，展示选项，执行选择

## 何时停止并寻求帮助

**当出现以下情况时立即停止执行：**
- 批次中间遇到阻碍（缺少引用文献，逻辑验证失败，指令不清楚）
- 计划有严重缺陷，无法开始
- 你不理解某个指令
- 验证反复失败

**寻求澄清而不是猜测。**

## 何时重新访问早期步骤

**在以下情况返回审阅（步骤 1）：**
- 合作伙伴根据你的反馈更新了计划
- 基本论证方法需要重新思考

**不要强行突破阻碍** - 停止并询问。

## 记住
- 先批判性地审阅计划
- 严格遵循计划步骤
- 不要跳过验证
- 当计划要求时引用技能
- 批次之间：只报告并等待
- 受阻时停止，不要猜测
