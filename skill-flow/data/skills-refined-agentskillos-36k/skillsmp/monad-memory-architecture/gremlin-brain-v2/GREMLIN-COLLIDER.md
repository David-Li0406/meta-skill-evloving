### SCALE GAME [Test at Extremes]
```
RULE: Extremes expose fundamental truths hidden at normal scales

| Dimension | Test At | Reveals |
|-----------|---------|---------|
| Volume | 1 vs 1B items | O(n) complexity |
| Speed | Instant vs 1 year | Async requirements |
| Users | 1 vs 1B | Concurrency limits |
| Duration | ms vs years | Memory leaks |
| Failure | Never vs Always | Error adequacy |

PROCESS:
1. Pick dimension that could vary
2. Test 1000x smaller
3. Test 1000x bigger
4. Note what breaks
5. Note what survives (that's fundamental)
```

### COLLISION ZONE [Force Conceptual Collisions]

RULE: Revolutionary insights from forcing unrelated concepts together

PROCESS:

### EDGE CASE HUNTING [The Gremlin Specialty]
```
ALWAYS ASK:
- What if empty? null? undefined? NaN? Infinity?
- What if negative? MAX_INT+1? 0? -0?
- What if emoji? RTL text? Zero-width joiners?
- What if concurrent? Race condition? Deadlock?
- What if midnight + timezone change?
- What if user inputs SQL? HTML? JavaScript?
- What about the cursed number that's technically valid?

RED FLAGS:
"Users will never do that" → They will
"This always returns true" → Does it?
"It's always been done like this" → We'll do it different
```
