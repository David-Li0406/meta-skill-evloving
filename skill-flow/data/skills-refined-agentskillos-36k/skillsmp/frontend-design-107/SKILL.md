This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## When to Use
- The user asks for a UI component, page, or web app.
- The task is frontend design-heavy and needs a strong aesthetic direction.

## Process
1. Clarify the context: purpose, audience, and technical constraints.
2. Choose a bold, specific aesthetic direction and commit to it.
3. Implement working code (HTML/CSS/JS or framework) that is production-grade, cohesive, and memorable.
4. Match implementation complexity to the aesthetic vision.

## Discovery (Ask Before Designing)
Ask these first unless the user says "skip questions":
- Audience: Who hires you most (homeowners, builders, architects)?
- Positioning: What does "high-end" mean to you (materials, finish, price)?
- Proof: What should we lead with (photos, reviews, licenses, warranty)?
- Taste: Any websites or brands you like or dislike?
- Content: What assets exist (project photos, before/after, testimonials)?

## Assume-and-Propose (If the Client Is Unsure)
If answers are vague or missing, propose 2 to 3 directions, each with:
- Name + 1-line mood
- Palette (2 to 4 colors)
- Typography style (display + body)
- Key layout idea
- Sample hero copy

## Design Thinking
- Purpose: What problem does this interface solve? Who uses it?
- Tone: Pick a clear direction (e.g., editorial, brutalist, retro-futuristic, luxury).
- Constraints: Framework, performance, accessibility.
- Differentiation: Define the one unforgettable element.

## Frontend Aesthetics Guidelines
- Typography: Use distinctive, characterful fonts; avoid generic defaults.
- Color & Theme: Cohesive palette; CSS variables; strong dominant + sharp accents.
- Motion: Favor a few high-impact animations (page-load, staggered reveals).
- Layout: Asymmetry, overlap, diagonal flow, grid breaks where appropriate.
- Backgrounds: Use gradients, textures, shapes, or patterns to build atmosphere.

## Local Research Mode (No Web Access)
When web search is unavailable, infer from the brief:
- Region cues (e.g., coastal -> light, airy, sun-bleached tones).
- Craft cues (e.g., tiling -> precision, material textures, detail shots).
- Buyer cues (e.g., high-end -> restraint, confidence, whitespace).

## Web Research Mode (If Permitted)
If the user approves web access:
- Scan 3 to 5 competitor sites for layout, palette, and messaging patterns.
- Identify 1 to 2 gaps or opportunities to differentiate.
- Summarize in 5 to 7 bullets before proposing directions.

## Hard Constraints
- Do not use generic, overused font stacks (Inter, Roboto, Arial, system).
- Avoid purple-on-white defaults and boilerplate layouts.
- Avoid repetitive "AI-looking" styling; each design should feel intentional.
- Vary aesthetics across outputs; do not converge on a single look.

## Output Expectations
- Provide working code.
- Be visually striking, cohesive, and purposeful.
- Use comments sparingly and only for non-obvious logic.
