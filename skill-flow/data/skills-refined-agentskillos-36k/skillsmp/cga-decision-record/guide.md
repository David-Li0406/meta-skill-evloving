# cga-decision-record 詳細ガイド

## 概要

対話形式でADR/DDR/AIDRを作成する。AskUserQuestionを使って順番に質問し、
回答をもとにDRファイルを自動生成する。

## 対話フロー

### Step 1: 基本情報を聞く

以下を順番に質問する（AskUserQuestionを使用）:

1. **種類**: DDR（ドメイン）/ ADR（アーキテクチャ）/ AIDR (AI駆動ワークフロー)
2. **決定事項**: 何を決めたか（1文で）
3. **選択肢**: 検討したA、B、C...の選択肢
4. **選んだもの**: どれを選んだか
5. **不可逆性**: 1-10（1=後から再選択可能、10=取り返しがつかない）

### Step 2: 理由を深掘り

- なぜ他の選択肢ではなく、これを選んだか
- トレードオフは何か
- どんな状況が変われば再考するか

### Step 3: ファイル生成

`doc/decisions/{種類}-{YYYYMMDDHH}-{slug}.md` として保存

## 出力フォーマット

```markdown
# {種類}-{YYYYMMDDHH}: {決定事項}

| 項目 | 内容 |
|------|------|
| 日時 | {YYYY-MM-DD HH:mm} |
| 決定者 | {名前} |
| 種類 | {DDR/ADR} |
| 決定 | {選んだ選択肢} |
| 不可逆性 | {1-10}/10 |

## コンテキスト

{なぜこの意思決定が必要になったか}

## 選択肢

| 選択肢 | 概要 |
|--------|------|
| A | {説明} |
| B | {説明} |
| **C（採用）** | {説明} |

## 決定理由

なぜAでもBでもなくCを選んだか:

{理由の詳細}

## トレードオフ

{この決定で失うもの、リスク}

## 再考のトリガー

以下の状況が発生したら、この決定を見直す:

- {条件1}
- {条件2}
```
