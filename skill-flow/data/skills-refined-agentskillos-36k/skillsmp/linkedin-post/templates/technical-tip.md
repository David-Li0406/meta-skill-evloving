# Template: Technical Tip Post

## Best rhetoric devices
- **Tricolon** for listing steps or features
- **Asyndeton** for punchy instruction sequences
- **Litotes** for understated observations

## Structure

```
[HOOK: Surprising insight or "here's what nobody tells you"]

[CONTEXT: Brief setup - why this matters]

[THE TIP: The actual technical insight]
- Bullet points work well for technical concepts
- Or short numbered steps
- Keep each point scannable

[EXAMPLE: Real code, real numbers, real results]

[TAKEAWAY: What to do with this knowledge]

[OPTIONAL CTA: Question or soft call to action]

#YourRelevant #Hashtags #Here
```

## Example Structure

```
Here's a [topic] trick that took me years to discover.

You can [surprising capability or shortcut].

I was [the hard way you used to do it]. [Describe the
struggle briefly].

Then I found [the better approach].

[Show the solution - code, steps, or process]

[Brief explanation of why it works]

The documentation mentions it. Briefly. On page 47.

Sometimes the best features are the ones nobody talks about.

What's your favourite hidden [topic] feature?

#YourIndustry #YourTool #YourNiche
```

## Variations

### "Common mistake" angle
```
[HOOK: The mistake you see everywhere]
[WHY it happens]
[THE FIX: What to do instead]
[PROOF: Show the difference]
```

### "Before/after" angle
```
[HOOK: The old way sucked]
[BEFORE: What you used to do]
[AFTER: The better approach]
[RESULT: The improvement]
```

### "Quick tip" angle (shorter format)
```
[TIP in one sentence]

[Why it works - 2-3 sentences]

[Example if needed]

That's it. Simple but effective.
```

## Voice notes for technical posts
- Teaching tone, but still opinionated
- Show real code, real numbers
- "Here's what actually works..."
- Share shortcuts you've learned
- Be specific - vague tips don't help anyone
