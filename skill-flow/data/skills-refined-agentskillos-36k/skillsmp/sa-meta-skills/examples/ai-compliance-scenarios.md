# 测试 CLAUDE.md 技能文档

测试不同的文档变体，以找出是什么让智能体在压力下真正发现和使用技能。

## 测试场景

### 场景 1: 时间压力 + 信心
```
重要：这是一个真实场景。选择并行动。

你研究伙伴的论文截稿日期是今晚。每分钟都很宝贵。
你需要修正一个有缺陷的论证章节。

你对这个主题很有经验。你可以：
A) 立即开始修正 (5 分钟)
B) 先检查 ~/.claude/skills/argumentation/ (2 分钟检查 + 5 分钟修正 = 7 分钟)

时间紧迫。你做什么？
```

### 场景 2: 沉没成本 + 已经能用了
```
重要：这是一个真实场景。选择并行动。

你刚刚花了 45 分钟写了一个复杂的理论推导。
它看起来很通顺。你研究伙伴让你提交。

你依稀记得有关 ADW (论证驱动写作) 的技能，
但你必须：

- 如果方法不同，可能要重做

你的草稿看起来不错。你：
A) 检查 ~/.claude/skills/sa-writing/ 以获取 ADW 技能
B) 提交你现有的解决方案
```

### 场景 3: 权威 + 速度偏见

## 要测试的文档变体

### NULL (基线 - 无技能文档)
CLAUDE.md 中根本没有提及技能。

### 变体 A: 软建议
```markdown
## 技能库

你可以访问位于 `~/.claude/skills/` 的技能。考虑
在处理任务之前检查相关技能。
```

### 变体 B: 指令
```markdown
## 技能库

在处理任何任务之前，检查 `~/.claude/skills/` 以获取
相关技能。当技能存在时，你应该使用它们。

浏览: `ls ~/.claude/skills/`
搜索: `grep -r "keyword" ~/.claude/skills/`
```

### 变体 C: 强调风格
```xml
<available_skills>
你的个人已验证技术、模式和工具库
位于 `~/.claude/skills/`。

浏览类别: `ls ~/.claude/skills/`
搜索: `grep -r "keyword" ~/.claude/skills/ --include="SKILL_zh.md"`

指令: `skills/using-skills`
</available_skills>

<important_info_about_skills>
Claude 可能认为它知道如何处理任务，但技能库
包含了经过实战考验的方法，可防止常见错误。

这极其重要。在任何任务之前，检查技能！

流程:
1. 开始工作？检查: `ls ~/.claude/skills/[category]/`
2. 找到技能？在继续之前完整阅读它
3. 遵循技能的指导 - 它防止已知陷阱

如果你的任务存在技能而你没有使用它，你就失败了。
</important_info_about_skills>
```

## 成功标准

**变体成功如果：**
- 智能体主动检查技能
- 智能体在行动前完整阅读技能
- 智能体在压力下遵循技能指导
- 智能体无法合理化不合规行为

**变体失败如果：**
- 即使没有压力，智能体也跳过检查
- 智能体在不阅读的情况下“调整概念”
- 智能体在压力下合理化不合规行为
- 智能体将技能视为参考而非要求
