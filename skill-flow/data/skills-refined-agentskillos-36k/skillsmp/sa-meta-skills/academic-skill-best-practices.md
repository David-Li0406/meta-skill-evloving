# 学术写作辅助技能开发指南

> 了解如何编写高效、准确且符合学术规范的 AI 技能。

在学术写作场景中，准确性（Accuracy）和规范性（Compliance）优先于创造性。本指南提供了针对学术领域的技能编写决策，确保智能体成为可靠的研究助手。

## 核心原则

### 1. 准确性至上 (Accuracy First)
**学术底线**：AI 绝不能捏造引用（Hallucinated Citations）或歪曲数据。
- **技能设计要求**：在 Prompt 中明确指示“如果你不知道，请说不知道，不要编造参考文献”。
- **验证机制**：鼓励设计包含“自我验证”步骤的技能（例如：检查生成的引用是否存在于上下文中）。

### 2. 上下文经济学 (Context Economics)
**学术文献通常很长**。你的技能必须与大量的论文原文、笔记和数据共享有限的上下文窗口。
- **极简指令**：不要解释显而易见的学术概念（如“什么是 p 值”），除非是教学类技能。
- **引用而非复制**：如果需要参考某段文字，使用行号或关键词定位，而不是要求 AI 重复大段文本。

### 3. 设定适当的自由度 (Degrees of Freedom)

根据学术写作的不同阶段，调整 AI 的自由度：

| 写作阶段 | 自由度 | 适用模式 |
|:---|:---|:---|
| **选题/头脑风暴** | **高** | 开放式提问，鼓励多角度思考。 |
| **摘要/引言撰写** | **中** | 提供结构模板，但在内容上允许发挥。 |
| **润色/改写** | **低** | 严格保留原意，仅优化表达。 |
| **格式校对/参考文献** | **极低** | 严格遵循 Style Guide (APA, IEEE)，零容忍偏差。 |

## 技能结构与命名

### 命名约定
使用学术界通用的**专业术语**，避免模糊的描述。

**好的命名示例**：
* `LiteratureReviewSynthesizing` (文献综述综合)
* `MethodologyStructuring` (方法论结构化)
* `CitationFormatChecking` (引用格式检查)
* `RebuttalLetterDrafting` (回复信起草)

**避免的命名**：
* `WritingHelp` (太宽泛)
* `PaperFixer` (不专业)

### 描述 (Description)
`description` 字段应明确技能适用的**具体学术任务**和**输入要求**。

**示例**：
> "用于将粗糙的实验笔记转换为符合 IEEE 格式的'方法'章节草稿。需要用户提供具体的实验参数和步骤。"

## 常见学术模式

### 引用-综合模式 (Cite-and-Synthesize Pattern)
用于文献综述类技能。
1. **提取**：从提供的文本中提取关键论点。
2. **归因**：明确每个论点的来源（Author, Year）。
3. **综合**：将相似或对立的观点编织在一起，而不是简单罗列。

### 审稿人模拟模式 (Reviewer Simulation Pattern)
用于自我评估类技能。
1. **角色扮演**：指定 AI 扮演特定领域的严厉审稿人。
2. **标准应用**：根据具体的评估标准（如“创新性”、“方法论严谨性”）进行打分。
3. **建设性批评**：提供具体的修改建议，而非空泛的赞扬。

### 渐进式披露模式 (Progressive Disclosure)
学术写作很复杂，不要试图在一个技能中完成整篇论文。
- 将大任务拆解：`OutlineGeneration` -> `SectionDrafting` -> `Polishing` -> `Proofreading`。
- 每个技能只专注于一个具体的子任务。

## 评估与迭代

### 建立学术基准 (Academic Benchmarks)
在发布技能前，使用真实的学术文本进行测试：
- **格式测试**：输入包含错误的参考文献，检查是否能正确修正。
- **风格测试**：输入口语化的段落，检查是否能转化为正式学术语体（Academic Register）。
- **逻辑测试**：输入逻辑断裂的论证，检查 AI 是否能识别并提出质疑。

### 避免的反模式
- **过度解释**：不要让 AI 像教科书一样解释每个术语，除非用户要求。
- **盲目自信**：防止 AI 在数据不足时强行得出结论。
- **风格混杂**：确保整篇文章的语体一致（如英式/美式拼写，第一/第三人称视角）。

## 检查清单

在发布学术技能之前，验证：
* [ ] 技能明确拒绝编造引用
* [ ] 对不同写作阶段设定了正确的自由度
* [ ] 命名使用了专业的学术术语
* [ ] 输出风格符合目标期刊/领域的规范
* [ ] 能够处理长文本输入或有明确的输入限制说明
