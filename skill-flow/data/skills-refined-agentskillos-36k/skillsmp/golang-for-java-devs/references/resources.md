# Resources

For days when exercises feel too heavy.

Reading or watching counts as learning. You don't have to write code every session.

---

## Books

### Start Here

**"The Go Programming Language"** by Donovan & Kernighan
- The definitive Go book, written by legends
- Comprehensive but approachable
- Skip around. Recommended chapters for Java devs:
  - Chapter 1: Tutorial (quick overview)
  - Chapter 5: Functions (error handling patterns)
  - Chapter 7: Interfaces
  - Chapter 8: Goroutines and Channels
- Why it helps: Shows *why* Go is designed the way it is

### When You're Ready

**"Concurrency in Go"** by Katherine Cox-Buday
- Deep dive on goroutines, channels, patterns
- Good for when simple concurrency isn't enough
- Skip if you're not doing heavy concurrent work yet

**"Let's Go"** and **"Let's Go Further"** by Alex Edwards
- Practical web development with Go
- Builds real applications step by step
- Skip if you prefer learning patterns over projects

### Deep Dives (Optional)

**"100 Go Mistakes and How to Avoid Them"** by Teiva Harsanyi
- Common pitfalls and how to fix them
- Good reference when things go wrong
- Skip until you've written enough Go to make mistakes

## Online (Free)

### Quick Reference

**Tour of Go**
- https://go.dev/tour/
- Interactive tutorial in browser
- Good for: hands-on syntax refresher

**Go by Example**
- https://gobyexample.com/
- Annotated code examples for every concept
- Good for: "how do I do X in Go?"

**Effective Go**
- https://go.dev/doc/effective_go
- Official style and idiom guide
- Good for: understanding idiomatic patterns

### Documentation

**Go Standard Library**
- https://pkg.go.dev/std
- Comprehensive stdlib docs
- Good for: discovering what's built-in

**Chi Router**
- https://go-chi.io/
- Your HTTP router of choice
- Good for: routing patterns and middleware

**go-playground/validator**
- https://pkg.go.dev/github.com/go-playground/validator/v10
- Request validation library
- Good for: struct validation tags

### Videos

**JustForFunc** (YouTube) by Francesc Campoy
- Deep Go concepts explained well
- Good for: understanding internals

**GopherCon Talks** (YouTube)
- Conference talks from Go experts
- More advanced, architecture-focused
- Good for: understanding *why* Go does things

**Ardan Labs** (YouTube)
- Practical Go training
- Good for: professional Go patterns

## Tools

**gopls** (Go Language Server)
- Built-in to most IDEs
- Autocomplete, go-to-definition, refactoring
- Good for: productive coding

**golangci-lint**
- Meta-linter with many checks
- Catches common mistakes
- Good for: code quality

**air**
- Live reload for development
- Rebuilds on file changes
- Good for: faster iteration

## Reading Strategy

### Low Energy
- Scan Go by Example for a topic you already know
- Watch a short YouTube video
- Read one page from Effective Go

### Medium Energy
- Follow a Tour of Go section
- Read a chapter from "The Go Programming Language"
- Watch a GopherCon talk

### High Energy
- Do exercises instead of reading
- Reading is for recovery days

---

Consuming content is still progress. Some days, that's enough.
