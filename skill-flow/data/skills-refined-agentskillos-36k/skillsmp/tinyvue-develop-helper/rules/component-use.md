# TinyVue组件的使用

## 组件索引

所有的组件名称都可以从 [menus.js](../menus.js) 文件中获得。在 `cmpMenus` 变量中，存放的组件的目录，它包含着二级数据，第一级是组件的类型，第二级数据才是真正的组件信息。

第二层的每一项数据的`name` 记录着组件的名称， `namecN` 记录着组件的中文名称， `key` 记录着组件的小写连字符格式的组件名。通过 `key` 属性可以在`apis`中找到相应的组件API文档， 在`demos`中找到相应的组件示例的文件夹。

`cmpMenus`的数据格式示例：

```
 {
    'label': '基础',
    'labelEn': 'Basic',
    'key': 'cmp-frame-style',
    'children': [
      { 'nameCn': '按钮', 'name': 'Button', 'key': 'button' }]
 }
```

上述数据描述了：在`基础`类别的组件中`Button`组件，它的 key 是'button' 。

### 组件的API文档索引

在`../apis` 目录中，每一个文件对应一个组件的API, 其文件名对应于组件英文名，比如： `action-menu.js` 就是对应着 `ActionMenu` 组件。

每一个API文件中，`apis`属性是一个数组，以描述复杂组件可能包含多个组件，比如 Form 和 FormItem， 这时每一个数组项就对应着一个组件的 API 信息。`types`属性也是一个数组，描述着当前组件用到的一些`typescript`类型的描述。

一个组件的API信息包含：name, props,events,methods,slots等属性。

在组件的API信息的`props`描述了组件有哪些属性，`events`描述了组件有哪些事件，`methods`描述了组件的实例上有哪些方法 ，`slots`描述了组件有哪些插槽。

在这些描述信息中，`name`是相应的属性值，`type`是相应的属性类型， `desc`属性中包含中英文的描述，引用时必须引用中文的描述。`pcDemo`属性指向了这个属性关联着的demos示例文件，指向 `./demos/{组件名}/{pcDemo指向名称}.vue` 这个具体的demo源码。`typeAnchorName`属性，指向了它类型的详细接口定义，通常放在`types`数组中。

### 组件的示例索引

在`../demos`目录中，每一个文件夹对应一个组件的全部示例，其中文件名规则是：

- `${组件名}/webdocs/${组件名}.js` :组件的示例索引文件，其中`demos`属性存放着该组件所有的示例目录，记录着示例的说明和示例指向的示例源码。 每一个示例信息中， `demoId` 是示例的唯一id, `name`是示例的名字， `desc`是示例的详细说明， `codeFiles` 记录着该示例的源码位置。
- `${组件名}/${demoId}.vue` Vue的选项式语法的示例源码。
- `${组件名}/${demoId}-compossition-api.vue` Vue的组合式语法的示例源码。

示例：
[button的示例索引](../demos/button/webdoc/button.js) 存放着`button`组件的所有示例。

## 严格查找流程，严禁跳过任意阶段

查找组件的API和示例时，要严格遵循下面的查找阶段：

1. **组件名查找阶段** ：从`../menu.js`中找到适合的组件名称
2. **查找API信息阶段**: 通过组件名称，在`../apis`中查找组件的 API文档
3. **示例查找阶段**： 通过组件名称，在`../demos`中查找组件的示例源码。首先要查看`basic-usage.vue`的基础用法，再查看其它的示例用法。查找到准确的示例后，就不要再查找其它示例。

### 查找约束规则

1. 要根据组件名进行关联查询API和示例，严禁使用其它开源库的属性，来猜测TinyVue组件库的用法。
2. 读取属性描述或示例描述时，必须使用它的中文描述信息，通常是在 `zh-CN`的属性中描述的，一定要忽略`en-US`描述信息。
3. 查找不到相应的属性或示例时，不得自行推断或替换其它信息。
4. 遇到错误时，不得自行推断或替换其它信息。
