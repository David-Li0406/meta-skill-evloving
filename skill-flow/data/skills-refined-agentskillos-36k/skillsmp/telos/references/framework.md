# TELOS 框架详解

TELOS是希腊词，意为"目的"或"终点"。这套框架将模糊的"你是谁、你想要什么"结构化成AI可理解的上下文。

## 核心理念

### 通用算法（Universal Algorithm）

每次AI帮你做事，本质上都在执行：

```
当前状态 → 理想状态
```

- 当前状态：你现在在哪里
- 理想状态：你想去哪里
- 中间过程：怎么到达

**TELOS就是在定义"理想状态"。**

没有TELOS，AI只能给通用答案。
有了TELOS，AI知道"对你来说"的最佳答案是什么。

### 北极星效应

每次你问AI，它不是在真空中回答，而是知道：
1. 这个问题和你的哪个目标相关
2. 你在哪个项目的语境下问的
3. 你的障碍是什么（回答要避开这些坑）

---

## 六层结构

### 1. Problems（问题）

**核心问题**：什么让你不爽？

不是"你的目标是什么"，而是"什么让你不爽"。人的很多动力都来自这里。

- 你想减肥，是觉得"我现在的状态不对"
- 你想做产品，是因为"现有的解决方案很烂"

**要求**：具体。可以是个人问题，也可以是世界问题。

**示例**：
- 我父母每人打三份工，没时间陪我，这导致了什么
- 现在的漏洞管理工具根本不懂工程团队怎么工作
- 大部分人觉得自己的想法不值得分享

### 2. Goals（目标）

**核心问题**：你想改变什么？

不是"你想要什么"，而是"你想改变什么"。这是从问题到方向的转化。

**示例**：
- 问题：大部分人觉得自己只是打工的，不是创造者
- 目标：激活更多人，让他们意识到自己也有值得分享的想法

目标应该贯穿你所有的项目。

### 3. Challenges（挑战）

**核心问题**：什么在阻止你？

这是很多人跳过的一步，但特别关键。如果你不明确说出障碍，AI就没法帮你绕过去。

**障碍类型**：
- **技术的**：我不会编程
- **资源的**：我没时间
- **心理的**：我觉得自己不够格
- **系统的**：现有工具太复杂

**示例**：
- 想让更多人用上个人AI系统
- 障碍：大部分人不是技术背景，命令行对他们来说太难了
- 所以设计要考虑这个，要让非技术人员也能用

### 4. Strategies（策略）

**核心问题**：你打算怎么做？

从"想要"到"行动"的桥梁。不是具体任务清单，是方法论层面的。

**示例**：
- 从开源项目开始，降低门槛
- 用Markdown文件而不是数据库，保证可移植性
- 先解决自己的问题，再推广给别人
- 持续分享过程，激活其他人

### 5. Projects（项目）

**核心问题**：你现在在做什么？

列出当前项目，每个项目都应该对应回目标和策略。

**示例**：
- PAI：个人AI基础设施框架
- Fabric：AI工具集
- Unsupervised Learning：周报
- 客户咨询工作
- 漏洞赏金研究

### 6. Workflows（工作流）

**核心问题**：你每天实际在做什么？

这个特别重要，因为AI要帮你，就得知道你的日常是什么样的。

**示例**：
- 早上散步时用Limitless录音捕捉想法
- 回来后让AI整理这些想法
- 用"议会辩论"功能让多个AI角色挑战这个想法
- 实时编辑，形成文章
- 一键发布到X和LinkedIn

## 进化机制

TELOS不是静态的，它会进化。

当你：
- 看了一个YouTube视频
- 读了一本书
- 学了新技术

可以让AI：
1. 读取这个新知识
2. 对照你的TELOS
3. 建议如何升级整个系统

**示例**：读了本关于修辞学的书，AI会说：

> 根据这本书的原则，我建议：
> 1. 在你的写作技能里加入这些修辞手法
> 2. 调整博客工作流，加入修辞检查步骤
> 3. 更新你的风格指南

**AI在帮AI自己变得更懂你。**

## 实操步骤

### 第一步：倾倒（Dump）

找个安静的时间，用语音或文字，回答这些问题：
- 什么让你不爽？（个人的、工作的、世界的）
- 如果你有魔法棒，你会改变什么？
- 你现在在做什么？为什么做？
- 什么在阻止你做得更好？

### 第二步：结构化

把这些内容按TELOS框架整理成六个部分。

### 第三步：加载到系统

每次启动Claude Code，它会读取这个TELOS文件。

### 第四步：迭代

用一段时间后，你会发现：
- 有些目标变了
- 有些障碍消失了
- 有些新问题出现了

定期更新TELOS。建议至少每个月review一次。

## 核心洞察

> TELOS最重要的作用不是"让AI更聪明"，而是**让你更清楚**。
> 
> 很多人其实不知道自己想要什么。TELOS逼你做这件事。
> 
> 一旦你理清楚了，AI只是放大器。
> 
> **没有TELOS，AI是个聪明的陌生人。**
> **有了TELOS，AI是个懂你的助手。**
