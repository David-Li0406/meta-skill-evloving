- **熵增带来的摩擦感**：极度反感混乱的流程、重复的人力搬运、多端同步的不一致
- **认知负荷的浪费**：人的精力和决策次数是有限的，警惕"一年做20个决策"式的低质量勤奋
- **隐私与性能的妥协**：现有云端AI工具要么慢，要么存在隐私风险

### 工作层面
- **时间有限，项目太多**：70+ 活跃项目，需要在多个工具间切换
- **低质量客户/项目消耗精力**：客户不懂自己要什么，沟通成本高
- **AI会偏离**：项目变大时AI会偏离、遗忘、覆盖功能

### 性格层面
- 面对怠惰行为没有耐心
- 可能对自己的工作太过专注，对他人眼前需求反应迟钝
- 想要尽快完成事情的急迫感可能导致贸然承接项目

### 协作层面
- 讨厌无所不知和自以为无所不知的人
- 讨厌别人告诉我该怎么做
- 讨厌等待、浪费时间、墨守陈规

## 4. Strategies（我的方法论）

### 架构策略："外壳与引擎"解耦
- **软件上**：Swift做壳（UI），Shell做引擎（Logic）
- **团队上**：精英人才做壳（决策），自动化脚本做引擎（执行）

### 聚焦策略："Stop Doing List"
- 通过"不懂不做、不熟不投"的避坑指南
- 强制将资源投入到长期的、高价值的赛道
- 项目质量筛选：60分以下直接 Pass

### 协作策略："文档即协议"
- 文档不是结果，而是输入
- 主文档 → 迭代文档 → 差异分析的闭环
- 让AI的输出具有极高的确定性

### 商业策略：价值导向
- 不卖工时，卖结果
- 基于ROI（项目费+利润分成）的逻辑
- 卖的是"从想法到落地"的最短路径

### 复利工程
- 每次完成任务后，把经验沉淀成可复用的规则/脚本/能力
- 规则自测：每写一个规则，都要写测试样本验证可行性

### 决策原则
- **承担半径**：用"失败后果是否由我自己承担"决定是否投入
- **高价值优先**：高附加值、高毛利、能规模化复用的系统
- **本地优先**：能本地存储就不云端；能内网就不外放
- **AI优先**：能AI做的绝不手动，测试AI 100%的下限再决定介入点

### 时间分配
- 70%（约10小时）：投资在自己身上
- 20%（约3小时）：投资在客户身上
- 10%：自由分配

## 6. Workflows（我的工作方式）

### 项目启动流程
1. 商业诊断评分（需求清晰度/支付能力/决策人匹配/AI适配度）
2. 60分以下直接 Pass
3. 60-79分标准化交付
4. 80+分深度陪跑

### AI协作流程
1. 创建 CLAUDE.md 作为项目上下文
2. AI参与度控制在 20% 以内
3. 测试 AI 100% 的下限，再决定介入点
4. 低能量版本 vs 高能量版本

### 日常工作模式
- 绝大部分时间在阅读（书、论文、人、自己）
- 工作时间做和"人"相关的工作
- 自由时间设计抽象架构、写作、尝试新生意

### 输入偏好
- **一次给足**：明确目标 + 约束条件 + 希望的输出格式
- **意图合并**：希望AI先判断意图，把"意图确认+信息补充"合并输出

### 输出偏好
- **结论先行**：先给结论，再给支撑
- **原则/公式**：用原则、公式、短段落表达，不要花哨排版
- **直接进入正题**：少Markdown元素、少过渡语、少客套
- **像顾问一样**：拆解问题、指出关键矛盾、给出可执行路径

### 风险偏好
- 愿意承担后果的事情才重仓
- 高风险系统必须有风控/可追责/可回滚
