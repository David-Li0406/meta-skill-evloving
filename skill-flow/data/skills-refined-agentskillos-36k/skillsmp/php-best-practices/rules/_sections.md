# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Security (security)

**Impact:** CRITICAL
**Description:** Security best practices to prevent injection attacks, XSS, CSRF, and other common PHP vulnerabilities.

## 2. Error Handling (error)

**Impact:** HIGH
**Description:** Robust error handling and exception management for maintainable and debuggable applications.

## 3. Performance (perf)

**Impact:** HIGH
**Description:** PHP performance optimizations including OPcache, autoloading, and memory-efficient patterns.

## 4. Type Safety (types)

**Impact:** MEDIUM-HIGH
**Description:** Leveraging PHP's type system for safer, more predictable code with better IDE support.

## 5. OOP Patterns (oop)

**Impact:** MEDIUM
**Description:** Object-oriented programming patterns for maintainable and testable code architecture.

## 6. PSR Standards (psr)

**Impact:** MEDIUM
**Description:** Following PHP-FIG standards for interoperability and consistent code style.

## 7. Testing (test)

**Impact:** MEDIUM
**Description:** Testing patterns and practices for reliable PHP applications.

## 8. Modern PHP (modern)

**Impact:** LOW-MEDIUM
**Description:** Leveraging PHP 8+ features for cleaner, more expressive code.
