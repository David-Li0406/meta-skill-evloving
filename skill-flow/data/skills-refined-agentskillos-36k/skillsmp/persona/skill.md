# Amazon Persona Writing Skill

Use this skill when creating comprehensive Amazon-style persona documents that guide product, engineering, marketing, and sales decisions. These personas embody Amazon's working backwards principles and customer obsession.

**Template:** See `docs/templates/persona-template.md` for the authoritative template structure.

## When to Use This Skill

- When defining target customer segments for new products
- When creating persona documentation for stakeholder alignment
- When converting customer research into actionable personas
- When preparing customer-focused documentation for leadership review

## Document Structure Summary

Create a 1,000-1,500 word document with all 17 sections in order:

1. **Metadata** (YAML header) - ID, name, type, segment, market, geo
2. **Snapshot** (table) - Role, seniority, company details, influence
3. **Customer Goals & Success Metrics** - Primary/secondary goals, KPIs
4. **Jobs-to-Be-Done** (3-5 JTBD statements in When/I want/So I can format)
5. **Pain Points & Triggers** - Ranked pain points, triggering events
6. **Behaviors & Habits** - Research channels, info sources, collaborators
7. **Decision Criteria & Objections** - Must-haves, evaluation rubric, objections
8. **Buying Committee/Influence Map** - DMU roles and concerns
9. **Journey Map** (compact) - Awareness through advocacy stages
10. **Messaging Guide** - Value prop, proof points, do/don't words, elevator pitch
11. **Channel Strategy** - Primary channels, content types, effectiveness
12. **Product Fit & Gaps** - Strengths and weaknesses for this persona
13. **KPIs & Leading Signals** - Fit indicators, tracking metrics
14. **Competitive Posture** - Alternatives, how we win
15. **Evidence & Confidence** - Data sources, quotes, confidence rating
16. **Anti-Persona** - Who this is NOT for
17. **Open Questions & Research Plan** - Unknowns, next research steps

## Writing Style

| Aspect | Guideline |
|--------|-----------|
| **Length** | 1,000-1,500 words |
| **Tone** | Neutral, professional, Amazonian |
| **Assumptions** | Mark with **(Assumption)** |
| **Specificity** | Avoid generic statements |
| **Action** | Every section must be actionable |

## Quality Checklist

- [ ] All 17 sections present in order
- [ ] YAML metadata complete
- [ ] JTBD in proper format (When/I want/So I can)
- [ ] Pain points ranked by severity
- [ ] Assumptions explicitly marked
- [ ] Evidence sources cited
- [ ] Anti-persona defined
- [ ] 1,000-1,500 words
