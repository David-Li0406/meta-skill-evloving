# 测试用例概览模板

使用此模板生成 `docs/devdocs/03-test-cases.md`

---

```markdown
# 测试用例：<功能名称>

## 1. 测试策略

### 1.1 测试类型

| 测试类型 | 目的 | 覆盖范围 |
|----------|------|----------|
| 单元测试 | 验证内部逻辑 | 核心业务函数 |
| 集成测试 | 验证组件协作 | 服务间调用、数据库操作 |
| E2E 测试 | 验证用户场景 | 完整用户流程 |

### 1.2 测试原则

- 测试用例从需求推导，不是从代码推导
- 每个验收标准至少有一个测试用例覆盖
- 优先覆盖 P0 功能

---

## 2. 覆盖率要求

| 测试类型 | 覆盖目标 | 要求 |
|----------|----------|------|
| 单元测试 | 核心业务逻辑 | 行覆盖率 ≥ 80%，分支覆盖率 ≥ 80% |
| 集成测试 | 组件协作 | 每个功能点至少 1 个 IT |
| E2E 测试 | 用户故事 | 每个 P0 用户故事至少 1 个 E2E |

---

## 3. 追溯矩阵

### 3.1 概览

| 指标 | 数量 | 覆盖率 |
|------|------|--------|
| 功能点 (F) | X | - |
| 用户故事 (US) | X | - |
| 验收标准 (AC) | X | - |
| 单元测试 (UT) | X | 100% AC |
| 集成测试 (IT) | X | 100% F |
| E2E 测试 (E2E) | X | 100% P0 US |

### 3.2 详细追溯

| 功能点 | 用户故事 | 验收标准 | 单元测试 | 集成测试 | E2E测试 | 状态 |
|--------|----------|----------|----------|----------|---------|------|
| F-001 | US-001 | AC-001 | UT-001 | - | E2E-001 | ✅ |
| F-001 | US-001 | AC-002 | UT-002 | - | E2E-001 | ✅ |
| F-001 | US-001 | AC-003 | UT-003 | IT-001 | E2E-001 | ✅ |
| F-001 | US-002 | AC-004 | UT-004, UT-005 | - | - | ✅ |
| F-001 | US-002 | AC-005 | UT-006 | - | - | ✅ |
| F-002 | US-003 | AC-006 | UT-007 | IT-002 | E2E-002 | ✅ |
| F-002 | US-003 | AC-007 | UT-008 | - | E2E-002 | ⚠️ |

### 3.3 状态说明

- ✅ 完整覆盖：有对应测试用例
- ⚠️ 部分覆盖：测试用例可能不足
- ❌ 未覆盖：无测试用例

## 5. 测试环境

### 5.1 测试框架

| 测试类型 | 框架 | 版本 |
|----------|------|------|
| 单元测试 | <Jest/Vitest/pytest> | <版本> |
| 集成测试 | <Jest/Vitest/pytest> | <版本> |
| E2E 测试 | <Playwright/Cypress> | <版本> |

### 5.2 执行命令

```bash
# 单元测试
npm test

# 单元测试 + 覆盖率
npm test -- --coverage

# 集成测试
npm run test:integration

# E2E 测试
npm run test:e2e
```
```
