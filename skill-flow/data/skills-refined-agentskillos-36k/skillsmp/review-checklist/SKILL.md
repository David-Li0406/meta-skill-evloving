# Code Review Helper Skill

コードレビュー時の包括的なチェックリストを提供します。

## レビューチェックリスト

### 1. コードの正確性
- ロジックが正しく実装されているか
- エッジケースが適切に処理されているか
- エラーハンドリングが適切か

### 2. 可読性とメンテナンス性
- 変数名、関数名が明確か
- コメントが適切か（過不足なく）
- コードの構造が理解しやすいか
- マジックナンバーや定数が適切に定義されているか

### 3. パフォーマンス
- 不要な計算や繰り返しがないか
- データ構造の選択が適切か
- メモリ使用量が妥当か

### 4. セキュリティ
- 入力値の検証が適切か
- SQLインジェクション、XSSなどの脆弱性がないか
- 機密情報がハードコードされていないか
- 認証・認可が適切に実装されているか

### 5. テスト
- ユニットテストが記載されているか
- テストカバレッジが十分か
- エッジケースのテストが含まれているか

### 6. 既存コードとの整合性
- コーディング規約に従っているか
- 既存のパターンやアーキテクチャに沿っているか
- 依存関係が適切に管理されているか

### 7. ドキュメント
- 必要な更新がされているか
- APIドキュメントが更新されているか
- READMEやCHANGELOGの更新が必要か

## 出力形式

上記チェックリストに基づいて評価し、以下の形式で出力してください:

1. **総合評価**: ✅ 問題なし / ⚠️ 要注意 / ❌ 要修正
2. **カテゴリ別評価**: 各カテゴリについて問題点を指摘
3. **改善提案**: 具体的な改善方法を提示
4. **優先度**: 緊急度の高い順に並べる

## 注意事項

- 建設的なフィードバックを心がける
- 問題点だけでなく、良い点も評価する
- 具体的なコード例を示して説明する
