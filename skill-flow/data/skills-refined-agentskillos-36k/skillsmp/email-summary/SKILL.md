# 技术支持邮件总结

## 概述

将技术支持邮件内容提取并总结为结构化的简体中文报告，供研发团队快速了解问题和排查进展。

## 使用场景

当用户请求类似以下任务时使用此技能：
- "总结这封技术支持邮件"
- "提取这封邮件的关键信息给研发看"
- "帮我整理一下这个技术问题的核心信息"
- "这个客户反馈的问题，帮我总结一下"

## 总结流程

### 1. 提取关键信息

从邮件中提取以下信息：
- 产品信息（型号、序列号、固件版本）
- 故障现象和症状
- 已尝试的排查步骤
- 环境背景和相关事件
- 待处理事项
- 参与人员（重要：区分内部人员和客户）

详细提取指南见 [extraction-guide.md](references/extraction-guide.md)

**人员识别关键规则**：
- @novastar.tech 域名 = 公司内部人员（技术支持、研发、销售）
- 非 @novastar.tech 域名 = 客户或第三方

### 2. 生成总结报告

使用标准格式输出总结，确保：
- 使用简体中文
- 技术术语保留原文（如HDMI、MCTRL4K等）
- 客观描述，不添加推测
- 结构清晰，便于研发快速浏览

模板格式见 [summary-template.md](references/summary-template.md)

### 3. 输出内容

按以下顺序输出总结：

1. **问题概述** - 一句话说明核心问题
2. **产品信息** - 型号、序列号、固件版本
3. **故障现象** - 具体症状列表
4. **已尝试的排查** - 测试项目和结果
5. **环境背景** - 使用场景和异常事件
6. **待处理事项** - 明确的下一步行动
7. **参与人员** - 客户、技术支持、销售人员（通过邮箱域名区分）
8. **当前状态** - 案例进展阶段

## 示例

用户输入：
```
请总结这封邮件：
[粘贴技术支持邮件内容]
```

输出：
```
### 问题概述
MCTRL4K控制器通过HDMI输入时无视频显示，接收卡指示灯闪烁但无图像。

### 产品信息
- 产品型号: MCTRL4K
- 序列号: 2ECF12729N0A10004499
- 固件版本: 1.3.00

### 故障现象
- 接收卡指示灯每秒闪烁一次
- HDMI信号无法被识别
- 前面板LCD无显示
- NovaLCT可正常显示纯色（接收卡功能正常）

### 已尝试的排查
- [ ] 多台视频源设备测试 - 无改善
- [ ] 多种分辨率测试 - 无改善
- [ ] DP线缆测试 - 无改善
- [ ] HDMI-DP线缆测试 - 无改善
- [x] NovaLCT纯色测试 - 正常显示

### 环境背景
上次活动期间发生电源浪涌，可能损坏控制器。

### 待处理事项
- **技术支持**: 分析故障原因，确认是否为硬件损坏
- **客户**: 可能需要返修服务

### 参与人员
- **客户**: customer@example.com (客户公司)
- **技术支持**: zhangyw1@novastar.tech (处理工程师)
- **销售**: sales@novastar.tech (抄送)

### 当前状态
等待技术支持反馈
```

## 注意事项

- 保持简洁，省略客套话和签名
- 区分症状和原因，只记录观察到的现象
- 对于邮件线程，按时间顺序合并信息
- 不确定的信息不要编造
- 技术术语保持原文，便于研发理解
