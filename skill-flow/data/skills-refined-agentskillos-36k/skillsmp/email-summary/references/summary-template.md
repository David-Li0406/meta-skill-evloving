# 技术支持邮件总结模板

## 输出格式规范

当总结技术支持邮件时，使用以下结构输出（简体中文）：

---

### 问题概述
[一句话描述核心问题，包括产品型号和主要故障]

### 产品信息
- **产品型号**: [产品型号]
- **序列号**: [序列号]
- **固件版本**: [固件版本]
- **其他硬件**: [相关硬件信息]

### 故障现象
[列出具体症状和表现，按严重程度排序]

### 已尝试的排查
- [ ] [测试项目1] - 结果
- [ ] [测试项目2] - 结果
- [ ] [测试项目3] - 结果

### 环境背景
[使用场景、相关事件（如电源浪涌、软件升级等）]

### 待处理事项
- **[责任人]**: [具体任务]
- **[责任人]**: [具体任务]

### 当前状态
[等待反馈 / 需要服务 / 待客户确认等]

---

## 模板使用说明

1. **问题概述**：简洁明了，研发能快速理解问题
2. **产品信息**：所有能识别产品的关键信息
3. **故障现象**：客观描述客户反馈的症状，不做推测
4. **已尝试的排查**：列出客户或技术支持已做的测试和结果
5. **环境背景**：可能相关的环境因素或事件
6. **待处理事项**：明确的下一步行动和责任人
7. **当前状态**：案例当前所处阶段

## 注意事项

- 使用简体中文输出
- 技术术语保留原文（如HDMI、MCTRL4K等）
- 保持客观，不添加推测性内容
- 省略不必要的客套话和签名信息
