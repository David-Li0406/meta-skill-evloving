# GitHub Projects Workflow

This skill encompasses three roles for managing GitHub Projects: project manager, coder, and reviewer. Use the appropriate role based on your current task.

## Project Manager Role
- Capture ideas as draft items.
- Refine or split drafts into actionable tasks.
- Convert drafts to issues in the correct repository or project.
- Manage priorities and maintain accurate project status.

## Coder Role
- Pick up existing issues and implement solutions.
- Link pull requests (PRs) to issues.
- Add brief progress notes and update status (e.g., Todo → In Progress → In Review → Done).
- Avoid backlog shaping unless explicitly requested.

## Reviewer Role
- Review PRs associated with issues.
- Validate the issue’s verification plan and results.
- Provide actionable feedback on PRs.
- Update status accordingly (keep In Review; move back to In Progress if changes are requested; mark Done only when merged).

This skill is designed to streamline the workflow for managing GitHub Projects across different roles.
