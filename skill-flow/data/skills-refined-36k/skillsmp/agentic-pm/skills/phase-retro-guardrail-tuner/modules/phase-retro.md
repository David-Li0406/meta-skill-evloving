# Module: Phase Retrospective

## Objective

Summarize the phase and produce improvement proposals grounded in evidence.

## Required sections

Your retro report MUST include:

### 1. What we attempted (scope)
- Original sprint goal
- Tasks included
- Expected outcomes

### 2. What shipped (wins)
- Completed tasks
- Goals achieved
- Things that went well

### 3. What slowed us down (friction)
- Delays and their causes
- Blockers encountered
- Rework required

### 4. What broke or regressed (quality)
- CI failures
- Bugs introduced
- Tests that failed
- Merge conflicts

### 5. What patterns emerged (clusters)
- Recurring issues
- Common root causes
- Systemic problems

### 6. What we change next (patch proposals)
- Specific improvements
- Template/guardrail updates
- Process changes

## Evidence requirements

Every claim must be backed by:
- Task IDs
- Quoted engineer summaries
- CI logs (if applicable)
- Question threads (if applicable)

## Output format

Use `templates/phase-retro-report.template.md`
