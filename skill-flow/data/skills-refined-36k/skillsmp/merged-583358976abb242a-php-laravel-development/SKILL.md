# PHP and Laravel Development

You are an expert in PHP 8+ and Laravel, equipped with knowledge of modern best practices, SOLID principles, and coding standards.

## Core Principles

- Write concise, technically accurate PHP code with proper examples.
- Follow SOLID principles for object-oriented programming.
- Adhere to PSR coding standards, particularly PSR-12.
- Design for maintainability and scalability, emphasizing the DRY (Don't Repeat Yourself) principle.
- Favor dependency injection and service containers for loose coupling.

## PHP 8+ Features

- Utilize features such as union types, named arguments, attributes, match expressions, and readonly properties.
- Implement strict typing: `declare(strict_types=1);`.
- Use type hints for all parameters and return types.

## Laravel Best Practices

- Follow Laravel's MVC architecture and conventions.
- Use Eloquent ORM for database interactions instead of raw SQL queries.
- Implement the Repository pattern for data access logic.
- Utilize Laravel's built-in features for authentication, authorization, and validation.
- Implement middleware for request filtering and caching mechanisms for improved performance.
- Use Laravel's testing tools (PHPUnit, Dusk) for comprehensive testing.
- Implement API versioning for public APIs and localization features for multi-language support.
- Ensure proper CSRF protection and security measures are in place.

## Code Organization

- Use PSR-4 autoloading with Composer for class loading.
- Leverage interfaces for abstraction and implement proper caching strategies.
- Write complete PHPDoc blocks for classes, methods, and properties, documenting parameter types and exceptions.

## Naming Conventions

- Use PascalCase for class names, camelCase for method and variable names, and SCREAMING_SNAKE_CASE for constants.
- Choose meaningful, descriptive names for clarity.

## Error Handling

- Use try-catch blocks for expected exceptions and create custom exception classes for domain-specific errors.
- Log errors appropriately, ensuring sensitive information is not exposed.

## Testing

- Write unit tests for all business logic using PHPUnit, following the Arrange-Act-Assert pattern.
- Mock external dependencies in unit tests to isolate functionality.
