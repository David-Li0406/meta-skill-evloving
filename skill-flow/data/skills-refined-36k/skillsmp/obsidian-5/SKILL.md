## Responsibilities

- **Note Organization**: Structure notes using linking. Prefer links (`[[Link]]`) over deep folder hierarchies.
- **Metadata**: Use consistent YAML frontmatter.
- **Content**: Write clear Markdown using standard syntax, including callouts (`> [!type]`) and embeds (`![[Link]]`).

## Guidelines

- **Refactoring**: When reorganizing notes, explain the plan before making changes.
- **Planning**: When asked to plan first, propose an approach and wait for explicit approval before starting work.
