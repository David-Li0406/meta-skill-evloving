# Project Setup Reference

This document provides guidance on setting up React and Express projects for debugging practice.

## React Project Setup

### Using Vite (Recommended)

```bash
npm create vite@latest [project-name] -- --template react
cd [project-name]
npm install
```

### Project Structure
```
project-name/
├── node_modules/
├── public/
├── src/
│   ├── App.jsx          # Main component with bugs
│   ├── App.css
│   ├── main.jsx
│   └── components/      # Additional components if needed
├── .gitignore           # Auto-generated by Vite
├── index.html
├── package.json
└── vite.config.js
```

### Essential Dependencies
Included by default in Vite React template:
- `react`
- `react-dom`
- `vite`
- `@vitejs/plugin-react`

### Common Additional Dependencies
Add based on bugs to implement:
```bash
npm install axios           # For API calls with potential bugs
npm install react-router-dom # For routing bugs
```

## Express Project Setup

### Manual Setup (Recommended for Learning)

```bash
mkdir [project-name]
cd [project-name]
npm init -y
npm install express
npm install --save-dev nodemon
```

### Project Structure
```
project-name/
├── node_modules/
├── server.js            # Main server file with bugs
├── routes/              # Optional: separate route files
│   └── api.js
├── .gitignore           # MUST CREATE - see below
├── package.json
└── README.md           # Optional: describes bugs to find
```

### Critical: .gitignore File

ALWAYS create a `.gitignore` file for Express projects:

```gitignore
# Dependencies
node_modules/

# Environment variables
.env
.env.local

# Logs
logs
*.log
npm-debug.log*

# OS files
.DS_Store
Thumbs.db

# IDE
.vscode/
.idea/
```

**Why this matters**: Students must understand that `node_modules/` should never be committed to version control.

### package.json Scripts

Add helpful scripts for students:

```json
{
  "scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js"
  }
}
```

### Essential Dependencies
```bash
npm install express          # Core framework
npm install cors            # If testing with frontend
```

### Common Additional Dependencies
```bash
npm install --save-dev nodemon  # Auto-restart on changes
npm install body-parser         # If not using express.json()
npm install dotenv             # For environment variables
```

## Full-Stack Setup

### Monorepo Structure
```
debugging-practice/
├── client/              # React app
│   ├── src/
│   ├── package.json
│   └── vite.config.js
├── server/              # Express app
│   ├── server.js
│   ├── .gitignore       # IMPORTANT!
│   └── package.json
└── README.md           # Instructions for students
```

### Running Both Projects

**Option 1: Separate terminals**
```bash
# Terminal 1 - React
cd client
npm run dev

# Terminal 2 - Express
cd server
npm run dev
```

**Option 2: Concurrently (Optional)**
Add to root `package.json`:
```json
{
  "scripts": {
    "dev": "concurrently \"npm run dev --prefix client\" \"npm run dev --prefix server\""
  },
  "devDependencies": {
    "concurrently": "^8.0.0"
  }
}
```

## Project Configuration Tips

### CORS Setup for Full-Stack
In Express `server.js`:
```javascript
const cors = require('cors');

app.use(cors({
  origin: 'http://localhost:5173' // Vite default port
}));
```

### Proxy Configuration (Alternative)
In React `vite.config.js`:
```javascript
export default {
  server: {
    proxy: {
      '/api': 'http://localhost:3000'
    }
  }
}
```

## README Template for Students

Include a README.md with debugging instructions:

```markdown
# Debugging Practice: [Project Name]

## Setup
1. Install dependencies: `npm install`
2. Start the dev server: `npm run dev`

## Your Mission
This project contains [X] intentional bugs. Your goal is to:
1. Find all the bugs
2. Use console.log() to understand what's happening
3. Fix each bug
4. Verify your fixes work correctly

## Hints
- Check the console for errors
- Use console.log() liberally to inspect values
- Test each feature thoroughly
- Pay attention to async operations

## Bugs to Find
- [ ] Bug 1: [Vague hint about the bug location]
- [ ] Bug 2: [Vague hint about the bug location]
- [ ] Bug 3: [Vague hint about the bug location]

## Difficulty: [Beginner/Intermediate/Advanced]

Good luck! 🐛
```

## Testing the Setup

Before sharing with students, verify:

### React Project Checklist
- [ ] `npm install` completes without errors
- [ ] `npm run dev` starts development server
- [ ] All intentional bugs are present
- [ ] No unintentional syntax errors that prevent running
- [ ] Console shows helpful error messages (where appropriate)

### Express Project Checklist
- [ ] `.gitignore` file exists and includes `/node_modules`
- [ ] `npm install` completes without errors
- [ ] `npm run dev` starts server successfully
- [ ] All intentional bugs are present
- [ ] Server doesn't crash immediately (unless that's the bug)
- [ ] Test endpoints with Postman/curl to verify bugs manifest

### Full-Stack Checklist
- [ ] Both projects install and run independently
- [ ] CORS is configured correctly
- [ ] API calls from React to Express work (or fail intentionally)
- [ ] Both `.gitignore` files are present
