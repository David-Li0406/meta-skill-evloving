# X (Twitter) 运营安全规则

## 一、内容安全红线

### 1.1 绝对禁止的内容 (Zero Tolerance)

```
┌─────────────────────────────────────────────────────────────┐
│  ⛔ 政治相关                                                 │
│  - 任何国家/地区的政治讨论                                    │
│  - 选举、政党、政客相关内容                                   │
│  - 政策评论或批评                                            │
│  - 国际关系讨论                                              │
├─────────────────────────────────────────────────────────────┤
│  ⛔ 宗教相关                                                 │
│  - 宗教观点表达                                              │
│  - 宗教比较或评论                                            │
│  - 宗教相关争议                                              │
├─────────────────────────────────────────────────────────────┤
│  ⛔ 地域/种族相关                                            │
│  - 地域歧视或优越感                                          │
│  - 种族相关讨论                                              │
│  - 国籍相关争议                                              │
├─────────────────────────────────────────────────────────────┤
│  ⛔ 性别相关                                                 │
│  - 性别对立内容                                              │
│  - 性别歧视言论                                              │
│  - 争议性别议题讨论                                          │
├─────────────────────────────────────────────────────────────┤
│  ⛔ 其他敏感内容                                             │
│  - 暴力、色情内容                                            │
│  - 仇恨言论                                                  │
│  - 个人攻击                                                  │
│  - 未经证实的医疗/金融建议                                    │
│  - 阴谋论                                                    │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 允许的内容范围 (Safe Zone)

```
✅ 纯技术内容:
   - AI/ML 算法和原理
   - 编程语言和框架
   - 开源项目介绍
   - 技术教程和实践
   - 代码示例和 Debug
   - 论文解读 (纯技术层面)
   - 工具使用心得

✅ 行业观察 (谨慎):
   - AI 产品功能对比
   - 技术趋势分析
   - 公司产品发布 (仅事实描述)

⚠️ 灰色地带 (需审慎):
   - AI 伦理讨论 (仅限技术层面)
   - 行业裁员/招聘 (仅陈述事实)
   - 竞品对比 (避免贬低)
```

### 1.3 内容安全检查清单

发布前必须确认:

```
□ 是否涉及政治话题？ → 不发布
□ 是否涉及宗教话题？ → 不发布
□ 是否涉及地域/种族？ → 不发布
□ 是否涉及性别争议？ → 不发布
□ 是否可能引发争议？ → 重新考虑
□ 是否纯技术相关？ → 可以发布
□ 观点是否过于极端？ → 软化表达
□ 是否可能被误解？ → 补充说明
```

## 二、操作安全限制

### 2.1 频率限制 (Rate Limits)

基于 X 平台的反自动化机制，设定以下安全阈值:

| 操作类型 | 每小时上限 | 每日上限 | 最小间隔 | 随机延迟 |
|----------|-----------|----------|----------|----------|
| 点赞 | 30 | 200 | 3秒 | 3-10秒 |
| 转发 | 10 | 50 | 30秒 | 30-60秒 |
| 评论 | 15 | 80 | 60秒 | 60-120秒 |
| 发帖 | 3 | 15 | 1800秒 | - |
| 关注 | 10 | 50 | 60秒 | 60-120秒 |
| 取关 | 5 | 30 | 120秒 | 120-180秒 |
| DM | 5 | 20 | 300秒 | - |

### 2.2 新账号养号期

**账号年龄 < 30天**:
```
操作限制:
- 所有操作限制降至 50%
- 每日发帖不超过 5 条
- 每日评论不超过 20 条
- 避免大量关注操作
- 主要以浏览和点赞为主
```

**账号年龄 30-90天**:
```
操作限制:
- 所有操作限制降至 70%
- 逐步增加互动频率
- 可以开始有节奏的发帖
- 开始建立互动关系
```

**账号年龄 > 90天**:
```
操作限制:
- 可使用正常限制
- 但仍需保持自然节奏
- 持续监控账号状态
```

### 2.3 行为模式要求

```
模拟人类行为:
✓ 操作间隔随机化 (不要固定间隔)
✓ 浏览时间模拟 (点赞前先"阅读" 3-5秒)
✓ 滚动行为 (不要直接跳转到特定帖子)
✓ 偶尔的"犹豫" (加载页面后等待)
✓ 工作时间操作 (避免凌晨大量操作)
✓ 周末适当减少活动

避免的模式:
✗ 完全固定的操作间隔
✗ 连续快速点赞/评论
✗ 24小时不间断运行
✗ 完全相同的评论模板
✗ 同一时间每天发帖
```

## 三、账号安全措施

### 3.1 登录安全

```
推荐做法:
✓ 使用 Cookie 登录而非密码
✓ 定期更新 Cookie (每周一次)
✓ 使用固定 IP 或可靠代理
✓ 避免频繁切换设备/IP
✓ 开启两步验证 (手动登录时)

危险行为:
✗ 在多个设备同时登录
✗ 频繁切换 IP 地址
✗ 使用公共 WiFi 操作
✗ 分享账号凭据
```

### 3.3 异常检测与处理

**检测指标**:
```
监控以下情况:
- 操作成功率下降 (低于 90%)
- 出现验证码/人机验证
- 页面加载异常
- 功能被限制提示
- 粉丝异常流失
```

**应对措施**:
```
轻度限流 (操作偶尔失败):
→ 降低操作频率 50%
→ 增加操作间隔
→ 观察 2-4 小时

中度限流 (频繁失败):
→ 立即停止所有操作
→ 等待 6-12 小时
→ 手动登录检查状态
→ 逐步恢复操作

重度限流 (功能被禁):
→ 停止所有自动化
→ 等待 24-48 小时
→ 手动正常使用几天
→ 评估是否需要更换策略

账号被锁:
→ 按平台指引解锁
→ 分析被锁原因
→ 调整操作策略
→ 养号后再启动自动化
```

## 七、紧急情况处理

### 7.1 账号被限流

```
步骤:
1. 立即停止所有自动化操作
2. 不要尝试解锁或申诉
3. 等待 24 小时
4. 手动正常使用 2-3 天
5. 确认功能恢复后，以 30% 频率恢复自动化
6. 逐步增加到正常水平
```

### 7.2 账号被锁定

```
步骤:
1. 按 X 平台指引解锁
2. 通常需要手机验证
3. 解锁后 7 天内不要使用自动化
4. 分析被锁原因，调整策略
5. 以 20% 频率重新开始
```

### 7.3 内容被删除/警告

```
步骤:
1. 记录被删内容和原因
2. 检查是否触发敏感词
3. 更新敏感词库
4. 对类似内容加强审核
5. 如反复发生，考虑调整内容方向
```

## 九、每日安全检查

### 运营前检查

```
□ Cookie 是否有效？
□ 操作频率是否在限制内？
□ 是否有未处理的警告？
□ 账号状态是否正常？
```

### 运营后检查

```
□ 今日操作数据是否正常？
□ 是否有失败的操作？
□ 是否收到平台警告？
□ 内容是否全部通过审核？
```

### 每周安全审计

```
□ 更新 Cookie
□ 检查敏感词库
□ 审核过去一周的内容
□ 分析操作成功率
□ 检查账号健康度
□ 更新安全策略 (如需)
```
