# 发布前检查清单

## 📋 背景

根据 Antigravity 语音记录分析，发现多次出现"网站无法正常打开"、"功能失效"等基础问题。这些本应在发布前通过自检发现，但实际依赖用户发现后才修复，严重影响产品可信度和用户体验。

## 🎯 目标

建立系统化的发布前自检流程，确保：
- 基础功能正常可用
- 用户体验无明显问题
- 避免低级错误上线

## 📝 执行步骤

### P0级别（必须通过，否则不能发布）
- [ ] **网站能否正常访问**
  - 主页能正常加载
  - 无404/500错误
  - 资源文件正常加载

- [ ] **核心功能是否可用**
  - 主要功能可正常操作
  - 无JavaScript错误
  - API接口正常响应

- [ ] **数据是否正常加载**
  - 数据能正确显示
  - 无加载失败或空白页面
  - 图片/资源自动加载

### P1级别（强烈建议通过）
- [ ] **所有按钮/链接可点击**
  - 无死链
  - 点击后无异常
  - 按钮状态正确

- [ ] **表单输入/删除正常**
  - 输入框可正常输入
  - 删除键功能正确（不误删其他内容）
  - 无输入冲突

- [ ] **快捷键无冲突**
  - 不同场景快捷键不冲突
  - 符合用户预期
  - 有明确的快捷键说明

- [ ] **焦点管理正确**
  - 焦点移动符合逻辑
  - 删除后焦点自动移动到下一项
  - 无焦点丢失

### P2级别（建议检查）
- [ ] **不同浏览器兼容性**
  - Chrome、Safari、Firefox基本功能可用
  - 关键功能无浏览器特定问题

- [ ] **性能测试**
  - 首屏加载 < 3秒
  - 刷新响应 < 2秒
  - 无明显卡顿

- [ ] **移动端基本可用**
  - 主要功能在手机/平板可操作
  - 布局无严重错乱

## ✅ 验证标准

- P0项目必须100%通过
- P1项目至少90%通过
- P2项目根据实际情况决定

## ⚠️ 注意事项

1. **自检必须在发布前完成**，不能依赖用户发现
2. 每项检查都要记录结果，便于追溯
3. 发现问题立即修复，不能带着问题上线
4. 对于回归测试，重点关注曾经出问题的功能
