# Sections

## 1. App Router (app)

**Impact:** CRITICAL
**Description:** Server Components, layouts, loading states, and error boundaries. The foundation of Next.js App Router architecture that determines initial load performance and user experience.

## 2. Data Fetching (fetch)

**Impact:** CRITICAL
**Description:** Fetch caching, revalidation strategies, and streaming. Proper data fetching patterns are essential for performance and freshness.

## 3. Server Actions (actions)

**Impact:** HIGH
**Description:** Form handling, mutations, and optimistic updates. Server Actions enable progressive enhancement and simplified data mutations.

## 4. Rendering (render)

**Impact:** HIGH
**Description:** SSR vs SSG vs ISR, dynamic vs static rendering. Choosing the right rendering strategy significantly impacts performance and costs.

## 5. Middleware (middleware)

**Impact:** MEDIUM-HIGH
**Description:** Edge functions, redirects, and rewrites. Middleware runs before requests complete, enabling authentication, A/B testing, and internationalization.

## 6. Asset Optimization (assets)

**Impact:** MEDIUM
**Description:** next/image, next/font, and blur placeholders. Optimized assets improve Core Web Vitals and user experience.

## 7. Route Handlers (routes)

**Impact:** MEDIUM
**Description:** API endpoints and streaming responses. Route handlers provide backend functionality within Next.js.

## 8. Deployment (deploy)

**Impact:** LOW-MEDIUM
**Description:** Vercel configuration, edge runtime, and static export. Deployment settings affect performance, costs, and capabilities.
