## 概述

**目标**：通过标准化评估场景，量化 OHSpec SKILL 的性能和质量，指导优化方向。

**评估频率**：
- 每次重大优化后执行完整评估
- 每周执行快速评估（仅 simple 场景）
- 每月生成评估趋势报告

## 评估指标

### 1. 性能指标

**执行时间**：
```
执行时间 = 结束时间 - 开始时间
```

**Token 消耗**：
```
Token 消耗 = 总 token 使用量
Token 效率 = RFC 质量分数 / Token 消耗
```

**用户交互次数**：
```
交互次数 = 需要用户确认的次数
```

### 2. 质量指标

**RFC 完整性**：
```
完整性分数 = (实际章节数 / 必需章节数) × 100
```

**场景覆盖**：
```
场景覆盖分数 = (实际场景数 / 必需场景数) × 100
```

**DFX 覆盖**：
```
DFX 覆盖分数 = (实际维度数 / 8) × 100
```

**首次通过率**：
```
首次通过率 = 首次通过 precheck 的场景数 / 总场景数
```

### 3. 综合质量分数

```
质量分数 = (完整性分数 × 0.3) + (场景覆盖分数 × 0.3) + (DFX 覆盖分数 × 0.2) + (首次通过率 × 100 × 0.2)
```

## 执行流程

### 步骤 1：准备环境

```bash
# 清理旧数据
rm -rf .claude/ohspec/rfcs/*

# 确认 SKILL 版本
cat .claude/ohspec/SKILL.md | head -5
```

### 步骤 2：执行评估

对每个评估场景：

```bash
# 1. 记录开始时间
start_time=$(date +%s)

# 2. 执行 SKILL
/ohspec "$(cat evaluations/simple/eval-001-single-file.json | jq -r '.requirement')"

# 3. 记录结束时间
end_time=$(date +%s)

# 4. 计算执行时间
execution_time=$((end_time - start_time))

# 5. 提取 RFC 路径
rfc_path=$(ls -t .claude/ohspec/rfcs/ | head -1)

# 6. 运行 precheck
/ohspec:precheck $rfc_path

# 7. 记录结果
echo "场景: eval-001, 执行时间: ${execution_time}s, RFC: $rfc_path" >> evaluation-results.log
```

### 步骤 3：收集数据

```bash
# 提取 token 消耗
token_usage=$(cat .claude/ohspec/rfcs/$rfc_path/progress.json | jq '.token_usage.current')

# 提取用户交互次数
interaction_count=$(cat .claude/ohspec/rfcs/$rfc_path/progress.json | jq '.phases | length')

# 提取 precheck 结果
precheck_result=$(cat .claude/ohspec/rfcs/$rfc_path/progress.json | jq '.phases.precheck.status')
```

### 步骤 4：生成报告

使用报告模板生成评估报告（见下文）。

## 自动化脚本

### 评估执行脚本（伪代码）

```bash
#!/bin/bash
# evaluate.sh - OHSpec 评估执行脚本

EVAL_DIR="evaluations"
RESULTS_FILE="evaluation-results-$(date +%Y%m%d-%H%M%S).json"

echo "开始 OHSpec 评估..."
echo "{\"scenarios\": [" > $RESULTS_FILE

for complexity in simple medium complex; do
  for eval_file in $EVAL_DIR/$complexity/*.json; do
    eval_id=$(jq -r '.id' $eval_file)
    requirement=$(jq -r '.requirement' $eval_file)

    echo "执行场景: $eval_id"

    # 记录开始时间
    start_time=$(date +%s)

    # 执行 SKILL（需要手动执行或通过 API）
    # /ohspec "$requirement"

    # 记录结束时间
    end_time=$(date +%s)
    execution_time=$((end_time - start_time))

    # 收集结果
    # ... 提取 token、交互次数、质量分数等

    # 写入结果
    echo "{\"id\": \"$eval_id\", \"execution_time\": $execution_time, ...}," >> $RESULTS_FILE
  done
done

echo "]}" >> $RESULTS_FILE
echo "评估完成，结果保存到 $RESULTS_FILE"
```

## 注意事项

1. **环境一致性**：每次评估使用相同的环境和配置
2. **数据清理**：评估前清理旧数据，避免干扰
3. **结果记录**：详细记录每次评估的结果，便于趋势分析
4. **问题跟踪**：对失败场景进行根因分析
5. **持续改进**：根据评估结果调整优化方向
