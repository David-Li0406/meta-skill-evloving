# Domain Knowledge Template

Use this file to structure the domain references loaded for the skill.

## Suggested Sections
- Overview: what the domain covers and key terms
- Core concepts: definitions and relationships
- Systems: architectures, data flows, schemas
- Policies/constraints: rules, compliance notes, SLAs
- Playbooks: standard responses or actions
- FAQs: common questions and quick answers

## Checklist
- [ ] Content is indexed by headings for fast lookup
- [ ] Links to source docs or repos when possible
- [ ] Updated date noted somewhere to track freshness
- [ ] Redundancy avoided; keep single source of truth

## Anti-Patterns
- Scattering important details across multiple files without links
- Letting SKILL.md balloon with domain text
- Missing trigger words (project name, system names) in description
