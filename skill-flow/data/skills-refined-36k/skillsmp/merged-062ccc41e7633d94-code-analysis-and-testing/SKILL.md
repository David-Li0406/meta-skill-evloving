# /code-analysis-and-testing 스킬

이 스킬은 복잡한 코드 분석, 코드베이스 탐색, 그리고 엣지 케이스 테스트 생성을 서브에이전트에서 처리하여 메인 컨텍스트 오염을 방지하고 토큰을 절약합니다.

## 사용법

```
/analyze [분석 대상 및 질문]
/explore [질문]
/edge-test [파일 경로 또는 패턴]
```

## 예시

```
/analyze 번역기 성능 병목 지점
/explore 번역기 파이프라인 구조
/edge-test app/lib/youtube.ts
```

## 실행 규칙

1. Task tool 호출
2. `subagent_type` 지정: 
   - 분석: `"Plan"` 또는 `"Explore"`
   - 탐색: `"Explore"`
   - 테스트 생성: `"general-purpose"`
3. `thoroughness` 설정: 
   - 분석: `"very thorough"`
   - 탐색: 기본 `"medium"` (필요 시 `"very thorough"`로 변경)
4. 결과를 구조화된 형태로 반환 요청

## 프롬프트 템플릿

### 분석 요청

```
분석 요청: {user_query}

다음을 수행하세요:
1. 관련 코드 전체 탐색
2. 심층 분석 수행
3. 문제점/개선점 도출
4. 우선순위 제안
```

### 탐색 요청

```
사용자 질문: {user_query}

다음을 수행하세요:
1. 관련 파일과 코드를 탐색
2. 구조와 흐름을 파악
3. 핵심 정보를 요약하여 반환
```

### 엣지 케이스 테스트 생성 요청

```
엣지 케이스 테스트 생성 요청: {file_path}

## 1단계: 소스 파일 분석
- 파일을 읽고 모든 함수/컴포넌트 파악
- 각 함수의 파라미터 타입, 조건문, 분기 식별
- 에러 처리 로직 확인

## 2단계: 엣지 케이스 도출
각 함수에 대해 다음 케이스 식별:
- 빈 값, 경계값, 잘못된 타입, 특수문자, 매우 긴 입력, 유니코드
- 상태 관련 (초기, 로딩, 에러, 빈 데이터)
- 이벤트 관련 (키보드, 마우스)

## 3단계: 테스트 코드 생성
- 기존 테스트 파일이 있으면 해당 파일에 추가
- 없으면 새 테스트 파일 생성
```

## 적합한 작업

- 성능 분석
- 코드 구조 파악
- 엣지 케이스 테스트 보강
- 아키텍처 리뷰
- 리팩토링 계획
- 보안 검토
- 코드 품질 분석

## 부적합한 작업

- 단순 검색 (→ /search)
- 통합 테스트 (→ 수동 작성)
- E2E 테스트 (→ Playwright)
- 성능 테스트

```markdown
| 중간 | ... | ... |
| 낮음 | ... | ... |

## 액션 아이템

```

### 탐색 결과

- 서브에이전트가 탐색 완료 후 요약 반환
- 탐색 과정은 메인 컨텍스트에 포함되지 않음

### 엣지 케이스 테스트 결과

```markdown
## 생성된 엣지 케이스 테스트

### 파일: {test_file_path}

| 테스트명 | 카테고리 | 설명 |
|---------|---------|------|
| 빈 문자열 입력 | 경계값 | parseUrl('') 호출 시 |
| null 입력 | Null 처리 | parseUrl(null) 호출 시 |
| ... | ... | ... |

### 추가된 테스트 수: N개
### 예상 브랜치 커버리지 개선: +X%
```

## 효과

- 메인 컨텍스트 오염 방지
- 토큰 절약: 약 60-80%
- 체계적인 분석 및 테스트 결과
