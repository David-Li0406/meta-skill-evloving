# Update Issue and PR

This skill allows you to review and update the descriptions of issues and pull requests (PRs) as needed.

## Update Issue Workflow

### Parameters

| Parameter | Action |
|-----------|--------|
| `/.update-issue <issue_number>` | Process Issue #<issue_number> |
| `/.update-issue` | Retrieve 8 issues for confirmation and process them in bulk |

### Steps

1. **Retrieve Issues**: If no issue number is specified, fetch the latest 8 open issues sorted by last updated date.
   ```bash
   gh issue list --state open --limit 8 --json number,title,body,author,updatedAt | jq 'sort_by(.updatedAt) | reverse'
   ```

2. **Filter Unanalyzed Issues**: Extract issues that do not contain `※ Generated by Claude` in their body.

3. **User Confirmation**: Ask the user if they want to add analysis comments to the listed issues.
   ```
   以下のIssueに分析コメントを追記しますか？
   #1234 タイトル1
   #5678 タイトル2
   ...
   ```

4. **Bulk Processing**: If approved, analyze in parallel using a sub-agent and append comments to each issue.

### Single Issue Processing Workflow

1. **Retrieve Issue Content**:
   ```bash
   gh issue view <issue_number> --json body,title,author,assignees,comments
   ```

2. **Check for Analysis Comments**: Look for comments starting with `>` in the issue body.
   - **If present**: Review and update if necessary.
   - **If absent**: Analyze and append comments.

3. **Determine Information Sufficiency**: Assess if the issue contains enough information for implementation.
   - **Sufficient**: Append analysis comments to the body.
   - **Insufficient**: Post a comment asking for clarification.

4. **Propose Updates**: Confirm with the user before executing updates.

### Analysis Comment Format

- **Standard Pattern**:
   ```markdown
   > 要件が明確で、既存パターンを踏襲すれば実装できます。※ Generated by Claude
   ```

- **For Unclear Points**: Post a comment instead of appending to the body.
   ```bash
   gh issue comment {number} --body "@username 以下の点について確認をお願いします。
   - 質問1
   - 質問2
   > ※ Generated by Claude"
   ```

## Update PR Workflow

1. **Retrieve PR**:
   ```bash
   gh pr view --json number,title,body,url,isDraft
   ```

2. **Review Description**: Check if the PR description clearly states:
   - Overview: What is being achieved and how.
   - Changes: Specific changes made.
   - Verification: Checklist items present.

3. **Propose Updates**: If updates are needed, confirm with the user before making changes.
   - Present the proposed updates and ask for approval.

## PR Template

```markdown
このPRで何を実現するか、どのように実現するかを簡潔に記載

## 変更内容

- 変更点1
- 変更点2

## 動作確認

- [ ] 対象機能が正しく動作する
- [ ] 既存機能に影響がない
- [ ] ビルドが通る
```

## General Notes

- Ensure to confirm all updates with the user before executing any changes.
- Use the provided templates for consistency in documentation.
