# 内容结构设计

## 核心任务
基于确定的定位，设计内容的骨架：
1. 规划章节框架（整体结构）
2. 设计学习路径（知识递进）
3. 确定内容编排（每章详纲）
4. 评估可行性（时间、资源）

## 工作流程

### 第一步：结构模式选择

根据产品类型和定位，选择合适的结构模式。

**提问引导**：
```
基于您的定位，我们来确定结构模式：

常见结构模式：
A. 线性递进型（入门→进阶→高级）
B. 项目驱动型（围绕5-8个实战项目组织）
C. 问题解决型（围绕用户问题组织）
D. 技能树型（并行的技能模块，可灵活组合）
E. 混合型（主线 + 专题）

您的产品更适合哪种？
```

### 第二步：章节框架设计

设计完整的章节结构，确保：
- **开篇引入**：为什么学、学什么、怎么学（≤10%）
- **知识铺垫**：必要的基础知识（≤20%）
- **核心内容**：项目/技能的主体（≥60%）
- **进阶扩展**：可选的深入内容（≤20%）

### 第三步：学习路径设计

设计知识的递进关系和依赖，包括：
- 知识依赖图
- 难度曲线
- 多种学习路径（适应不同读者背景）

### 第四步：单章详纲

选择核心章节展开三级大纲，包含：
- 学习目标
- 内容明细（三级标题）
- 预估页数/字数
- 交付物清单

### 第五步：可行性评估

评估结构的可行性：
- 总字数/时长是否符合约束
- 工作量是否在时间范围内
- 资源需求是否满足
- 风险识别与应对

## 输出示例

```markdown
# 内容结构设计文档

## 结构模式
**选定**：项目驱动型 + 线性递进

## 章节框架
### 第一部分：基础篇（2章，2万字）
- 第1章：XXX（目标、内容、交付物）
- 第2章：XXX

### 第二部分：项目篇（5章，11万字）
- 第3-7章：五个递进项目

### 第三部分：进阶篇（2章，2万字）
- 第8-9章：工程化与职业发展

## 学习路径
[知识依赖图、难度曲线、多路径选择]

## 可行性评估
- 总字数：15-18万字 ✅
- 工作量：96天（4个月内）✅
- 风险：作者写作经验不足 ⚠️

## 下一步
进入策划案撰写（topic-proposal）
```

## 成功标准

- ✅ 结构模式清晰，符合定位
- ✅ 章节框架完整，覆盖核心内容
- ✅ 学习路径合理，难度递进
- ✅ 工作量评估准确，风险可控
