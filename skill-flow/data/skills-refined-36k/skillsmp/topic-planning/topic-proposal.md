# 策划案撰写

## 核心任务
整合前期所有成果，输出完整、规范、可操作的策划文档：
1. 汇总场景分析、调研、定位、结构的成果
2. 补充必要的执行细节（时间、资源、风险）
3. 输出标准化的Markdown文档
4. 确保策划案可直接用于决策和执行

## 策划案标准结构

```markdown
# [产品名称] 策划案

## 📋 项目概览
- 产品形态、主题、目标市场
- 项目阶段、时间节点
- 核心团队

## 🎯 定位与价值
- 目标用户画像
- 核心问题与解决方案
- 价值主张与差异化
- 市场定位

## 📊 市场分析
- 市场规模与趋势
- 竞品分析（TOP5深度分析）
- 用户洞察与痛点
- 市场机会

## 📚 内容规划
- 结构模式与章节框架
- 学习路径设计
- 核心章节详纲
- 内容策略与质量标准

## 📅 执行计划
- 里程碑与时间表
- 资源需求（人力、工具、预算）
- 质量保障措施
- 风险识别与应对

## 💰 商业规划
- 定价策略
- 销售预期
- 推广计划
- ROI预估

## 📎 附录
- 参考资料清单
- 竞品详细信息
- 用户调研数据
- 技术选型说明
```

## 工作流程

### 第一步：信息整合

收集前期成果：
- topic-scene-guide 的场景分析报告
- topic-research 的市场调研报告
- topic-positioning 的定位文档
- topic-structure 的结构设计文档

确认信息完整性，标注缺失项。

### 第二步：补充执行细节

添加可操作的执行信息：

**执行计划**：
- 详细的时间表（周/月级别）
- 每个阶段的交付物
- 关键里程碑和评审点

**资源需求**：
- 人力配置（作者、编辑、设计、技术审校）
- 工具和平台（开发环境、协作工具）
- 预算估算

**风险管理**：
- 识别TOP5风险
- 每个风险的应对预案
- 预留缓冲时间

### 第三步：撰写策划案

按照标准结构撰写完整文档，确保：
- **结构清晰**：使用Markdown标题层级
- **数据支撑**：关键判断有数据或调研依据
- **可视化**：使用表格、列表、图表增强可读性
- **可操作**：每个部分都有明确的行动指引

### 第四步：质量检查

自检清单：
- ✅ 定位清晰：一句话能说清楚产品是什么
- ✅ 差异化明确：与竞品的区别一目了然
- ✅ 结构合理：章节逻辑流畅，难度递进
- ✅ 可执行：时间、资源、风险都有具体方案
- ✅ 数据可信：市场数据有来源标注
- ✅ 格式规范：Markdown格式正确，排版美观

### 第五步：输出交付

生成最终文档：
```
[产品名称]-策划案-v1.0.md
```

同时生成：
- **一页纸摘要**（供快速决策）
- **执行检查表**（供项目管理使用）

## 特殊内容要求

### 市场数据标注
所有关键数据必须标注来源：
```markdown
- 市场规模：XX亿元（来源：艾瑞咨询2023年报告）
- 竞品销量：京东评价3.2万+（数据时间：2024-12-27）
```

### 风险预案
每个风险都要有具体应对措施：
```markdown
**风险1**：作者写作经验不足，可能影响进度
- **概率**：高
- **影响**：延期1-2个月
- **应对**：
  1. 提前进行写作培训
  2. 编辑深度参与，每周审稿
  3. 预留1个月缓冲时间
```

### 可视化呈现
关键信息使用表格或图表：
- 竞品对比表
- 时间甘特图（文字描述）
- 预算分配饼图（文字描述）

## 输出规范

1. **文件命名**：`[产品名称]-策划案-v[版本].md`
2. **Markdown规范**：
   - 一级标题用于大章节
   - 二三级标题用于内容组织
   - 代码块用于示例
   - 表格用于对比数据
3. **长度控制**：
   - 完整版：8000-15000字
   - 摘要版：1000-2000字
4. **附件要求**：
   - 将详细数据、调研原文放入附录
   - 主体内容聚焦决策要点
