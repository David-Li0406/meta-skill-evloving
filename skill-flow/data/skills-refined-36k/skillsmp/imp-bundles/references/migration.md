# Migration Workflow

To migrate an existing flake.nix to imp bundles:

## 1. Create nix/flake/ entry point

```nix
# nix/flake/inputs.nix
{
  nixpkgs.url = "github:nixos/nixpkgs/nixos-unstable";
  systems.url = "github:nix-systems/default";
  flake-parts.url = "github:hercules-ci/flake-parts";
  imp.url = "github:imp-nix/imp-nix";
}
```

```nix
# nix/flake/default.nix
inputs:
let
  inherit (inputs) nixpkgs flake-parts imp;
in
flake-parts.lib.mkFlake { inherit inputs; } {
  imports = [ imp.flakeModules.default ];
  systems = import inputs.systems;
  imp = {
    bundles.src = ../bundles;
    args = { inherit nixpkgs; rootSrc = ../..; };
    exports.enable = false;
    impShell.enable = true;
    flakeFile = {
      enable = true;
      coreInputs = import ./inputs.nix;
      description = "My project";
      outputsFile = "./nix/flake";
    };
  };
}
```

## 2. Replace top-level flake.nix

```nix
# flake.nix (auto-generated by imp)
{
  description = "My project";
  inputs = {
    self.submodules = true;
    flake-parts.url = "github:hercules-ci/flake-parts";
    imp.url = "github:imp-nix/imp-nix";
    nixpkgs.url = "github:nixos/nixpkgs/nixos-unstable";
    systems.url = "github:nix-systems/default";
    # __inputs from bundles appear here
  };
  outputs = inputs: import ./nix/flake inputs;
}
```

## 3. Create project bundle

Create `nix/bundles/<name>/default.nix`:

```nix
{
  __functor = _: { pkgs, buildDeps, rootSrc, ... }:
  let
    inherit (buildDeps.rust) rustPlatform;
  in {
    __outputs.perSystem.packages.default = rustPlatform.buildRustPackage {
      pname = "myproject";
      src = rootSrc;
      cargoLock.lockFile = rootSrc + "/Cargo.lock";
    };
  };
}
```

## 4. Add shared bundles as git submodules

```bash
git submodule add https://github.com/imp-nix/bundle.rust nix/bundles/rust
```

Create config override if needed:

```nix
# nix/bundles/rust.config.nix
{
  pname = "myproject";
}
```

## 5. Stage and test

```bash
git add -A
nix flake check
```

Nix flake only sees staged/committed files.

## CLI Subcommand Removal

If migrating agent docs from a CLI `agents` subcommand to skills:

1. Create skill structure:
   ```
   nix/bundles/<name>/skills/
     AGENTS.md
     <skill>/
       SKILL.md
       references/
   ```

2. Move agent docs to references, update headers

3. Remove CLI subcommand:
   - Delete command enum variant
   - Delete args struct
   - Delete match arm in run()
   - Remove include_str!() references

4. Update CLI help text to remove agent references
