# 任务分解参考指南

## 常用方法

### 1. WBS（工作分解结构）
- 自上而下分解
- 将项目分解为可交付成果
- 层级分明，便于管理

### 2. 思维导图法
- 可视化分解过程
- 激发创造力和联想
- 适用于复杂创意项目

### 3. 流程图法
- 展示任务依赖关系
- 识别关键路径
- 适用于流程性任务

## 最佳实践

### 分解原则
- 单一职责原则
- 可交付成果原则
- 可管理性原则
- 可验证性原则

### 依赖关系类型
- 前置依赖
- 并行关系
- 可选任务
- 循环依赖
