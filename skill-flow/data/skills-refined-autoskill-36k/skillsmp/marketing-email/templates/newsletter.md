# Template: Newsletter Email

## Purpose
Regular value delivery. Stay top of mind. Build trust.

## Best rhetoric devices
- **Tricolon** for listing insights
- **Anadiplosis** for flowing narratives
- **Litotes** for understated observations

## Structure

```
Subject: [Curiosity hook or clear value promise]

[HOOK: Interesting opening - story, observation, or surprising fact]

[MAIN CONTENT: The meat of the email]
- One clear insight or lesson
- Personal story if relevant
- Specific examples or code

[TAKEAWAY: What to do with this information]

---

[SOFT CTA: Question, invitation to reply, or gentle pointer]

[Your name]

P.S. [Optional secondary CTA or personal note]
```

## Subject Line Examples
- "The 30-second trick that took me years to find"
- "Why your [thing] is lying to you"
- "I deleted 500 lines of code yesterday"
- "The question that changed how I [do X]"
- "What I got wrong about [topic]"

## Example Newsletter Structure

```
Subject: The [topic] that made me look stupid

Last month, I [presented/delivered/showed] a [thing] to a [client/team].

[Time investment]. [Effort details].
Confident [outcome expectation].

Then someone asked: "[Question that exposed the flaw]"

I didn't know.

I'd [made assumption]. [Describe the assumption in detail].
Nice, neat, [convenient characteristic].

Real [systems/situations] don't work that way.

The fix was simple - [brief description of fix].
[Show the solution if relevant - code, process, approach]

But I'd missed it because I was optimising for "[wrong thing]"
instead of "[right thing]."

Now I start every [project/task] with one question:

[The better question to ask]

The [edge cases/exceptions/nuances] are where reality lives.

---

What assumptions have bitten you lately?
Hit reply - I read every response.

[Your name]

P.S. [Your topic] is covered in [your resource/offering].
Details here if you're curious: [link]
```

## Variations

### "Lesson learned" newsletter
```
[What happened]
[What I learned]
[How you can apply this]
```

### "Quick tips" newsletter
```
Three things I learned this week:

1. [Tip with brief explanation]

2. [Tip with brief explanation]

3. [Tip with brief explanation]

Which one's most useful for you?
```

### "Behind the scenes" newsletter
```
[What I'm working on]
[Why it matters]
[What's next]
[Invitation to follow along]
```

## Voice notes
- Conversational, not formal
- Share genuine insights, not filler
- Stories from real projects (anonymised if needed)
- Okay to admit uncertainty or mistakes
- End with engagement prompt (question or reply invitation)
