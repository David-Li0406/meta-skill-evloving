# 多视角思考框架 (Mental Models Toolkit)

基于斯科特·扬的25个职业思维工具、查理·芒格的多元心智模型理念，以及**马斯克的第一性原理**。

**核心理念**：
- **芒格**：知识不仅是事实，更是工具。每个职业都发展出独特的思维方式。
- **马斯克**：不要用类比思考，要回到最基本的物理/数学真相，从那里重新推导。

---

## 0) 硬约束

- **不是要用所有视角**：根据问题类型选择3-5个最相关的
- **每个视角要有具体产出**：不是空谈"从XX角度看"，而是产出具体的问题、洞察、行动
- **最终要综合**：多视角不是为了发散，而是为了更全面的收敛
- **承认局限**：标注哪些视角不适用于这个问题

---

## 1) 视角工具箱（25个，分6类）

### 分析型 - "这个东西是怎么运作的？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **工程师** | 能建模并计算吗？ | 精确测量、误差范围、模型验证 |
| **科学家** | 假设是什么？如何测试？ | 对照实验、变量控制、可证伪性 |
| **医生** | 症状是什么？诊断是什么？ | 症状→病因推理、基础率、排除法 |
| **会计师** | 关键比率是什么？ | 比率分析、趋势追踪、异常检测 |
| **数学家** | 能严格证明吗？ | 逻辑推导、边界条件、反例检验 |

### 创意型 - "这个东西能更好/更独特吗？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **艺术家** | 如果创造力是首要任务呢？ | 新颖性优先、打破常规、美学追求 |
| **设计师** | 形式如何暗示功能？ | 隐含交流、用户心理、可供性设计 |
| **建筑师** | 最终形态是什么样？ | 建模预演、缩放思考、长期视角 |
| **小说家** | 故事有意义吗？ | 叙事结构、角色弧线、因果逻辑 |

### 执行型 - "怎么把事情做成？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **企业家** | 能快速试错吗？ | 快速原型、MVP、迭代反馈 |
| **士兵** | 有纪律和常规吗？ | 标准流程、检查清单、零容错执行 |
| **水管工** | 能拆开看看吗？ | 动手拆解、从内部理解、实践验证 |
| **程序员** | 能自动化吗？ | 模式识别、抽象封装、算法思维 |

### 人性型 - "人会怎么想/怎么做？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **经济学家** | 人们对激励如何反应？ | 激励结构、博弈均衡、边际分析 |
| **销售员** | 他们真正想要什么？ | 需求挖掘、异议处理、说服逻辑 |
| **心理学家** | 认知偏见是什么？ | 偏见识别、行为预测、实验验证 |
| **政治家** | 人们会如何感知？ | 利益相关者分析、舆论管理、妥协艺术 |
| **演员** | 能真实感受吗？ | 方法演技、情绪调用、共情体验 |
| **教师** | 如何让别人理解？ | 知识诅咒克服、类比构建、渐进引导 |

### 批判型 - "什么是真的？什么被隐藏了？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **记者** | 事实核查了吗？ | 多源验证、追溯原始、质疑权威 |
| **哲学家** | 推到极限会怎样？ | 归谬法、思想实验、概念边界 |
| **黑客** | 底层真正发生了什么？ | 层级穿透、漏洞思维、系统边界 |
| **评论家** | 深层意义是什么？ | 密切关注、跨领域联系、元分析 |

### 策略型 - "对手/系统会怎么反应？"

| 视角 | 核心问题 | 思维工具 |
|------|----------|----------|
| **棋手** | 未来几步是什么？ | 走法模拟、决策树、对手建模 |
| **人类学家** | 这个文化如何运作？ | 沉浸观察、文化代码、局内人视角 |

## 2) 第一性原理分析（马斯克思维）

在多视角分析之前，先用第一性原理拆解问题。

### 什么是第一性原理？

**核心**：将复杂问题拆解到最基本、不可再分的要素，然后从这些基础真理重新构建，而不是依赖类比或"事情一直是这样做的"。

### 三步法

```
Step 1: 拆解 - 这个问题由哪些最基本的要素组成？
Step 2: 质疑 - 哪些"常识"其实是未经验证的假设？
Step 3: 重构 - 从基本要素出发，最优解是什么？
```

### 第一性原理提问清单

| 类别 | 核心问题 |
|------|----------|
| **成本拆解** | 这个东西的原材料/基本组成是什么？它们各自成本多少？ |
| **物理极限** | 物理定律允许的理论最优是什么？我们离它有多远？ |
| **假设质疑** | "必须这样做"的规则，是物理定律还是人为惯例？ |
| **类比陷阱** | 我是在用类比思考吗？类比的前提还成立吗？ |
| **基本命题** | 这个系统不可违背、不能删除的基本命题是什么？ |

### 经典案例

| 问题 | 类比思维 | 第一性原理 |
|------|----------|------------|
| 火箭太贵 | "火箭就是要这么贵" | 铝、钛、碳纤维、燃料各多少钱？→ 材料成本仅占2% |
| 电池太贵 | "电池技术就这样" | 钴、镍、锂、石墨各多少钱？→ 原料成本远低于售价 |
| 交易亏钱 | "市场就是难赚" | 我亏的钱去哪了？→ 手续费+滑点+追涨杀跌 |

### 输出格式

```markdown
## 第一性原理拆解

### 基本要素
这个问题拆解到最底层是：
1. [要素1]
2. [要素2]
3. [要素3]

### 质疑假设
以下"常识"其实是假设，不是真理：
- ❌ 假设1：...（实际上...）
- ❌ 假设2：...（实际上...）

### 物理/数学真相
不可违背的基本定律/约束：
- ✅ 真理1：...
- ✅ 真理2：...

### 重构方案
从基本要素出发，最优路径是：
...
```

## 3) 使用流程

### Step 0: 第一性原理拆解（可选但推荐）

对于重大决策或创新问题，先做第一性原理分析，再进入多视角分析。

### Step 1: 问题分类

收到用户问题后，先判断问题类型：

| 类型 | 特征 | 优先视角 |
|------|------|----------|
| **决策问题** | "该不该...？" | 经济学家、棋手、哲学家 |
| **诊断问题** | "为什么...？" | 医生、工程师、黑客 |
| **创造问题** | "如何做一个...？" | 艺术家、设计师、企业家 |
| **执行问题** | "怎么做到...？" | 士兵、程序员、水管工 |
| **人际问题** | "如何让他们...？" | 销售员、心理学家、教师 |
| **验证问题** | "这是真的吗？" | 记者、科学家、数学家 |

### Step 2: 选择视角（3-5个）

根据问题类型，选择最相关的视角。规则：
- 至少包含1个"对立"视角（如创意问题也要看执行视角）
- 优先选择与用户背景互补的视角
- 明确说明为什么选这些视角

### Step 3: 逐个视角分析

每个视角输出：

```
### [视角名] 的视角

**核心问题**：这个职业会问什么关键问题？

**洞察**：从这个角度，我看到了什么？
- 观察1
- 观察2
- 观察3

**盲区**：这个视角可能忽略什么？

**行动建议**：基于这个视角，应该做什么？
```

### Step 4: 综合分析

```
## 综合分析

### 视角共识
多个视角都指向的结论...

### 视角冲突
不同视角的矛盾点及如何处理...

### 关键盲区
所有视角都可能忽略的...

### 最终建议
综合所有视角后的行动方案...
```

## 3) 特定场景的推荐组合

### 交易/投资决策
- **经济学家**：激励结构和市场反应
- **医生**：市场"症状"诊断
- **棋手**：对手（市场参与者）会怎么走
- **士兵**：执行纪律
- **黑客**：表面叙事下的真实机制

### 产品/功能设计
- **设计师**：用户体验和隐含交流
- **企业家**：快速验证
- **销售员**：用户真正想要什么
- **程序员**：可实现性和自动化
- **评论家**：与竞品的差异化

### 人际沟通/说服
- **销售员**：对方真正的需求和顾虑
- **心理学家**：认知偏见和行为模式
- **演员**：情绪状态调用
- **教师**：如何让对方理解
- **政治家**：利益平衡和感知管理

### 问题排查/Debug
- **医生**：症状诊断和排除法
- **工程师**：建模和测量
- **水管工**：拆开看看
- **黑客**：底层发生了什么
- **科学家**：假设测试

### 战略规划
- **建筑师**：长期愿景
- **棋手**：多步推演
- **经济学家**：激励设计
- **哲学家**：边界和极限情况
- **企业家**：迭代验证

## 5) 使用示例

### 示例1：交易决策

**用户**：BTC突破10万美金了，我该追吗？

**选择视角**：
1. 经济学家 - 分析市场参与者的激励
2. 医生 - 诊断市场"症状"
3. 棋手 - 推演接下来几步
4. 士兵 - 执行纪律检查
5. 黑客 - 表面叙事下的真实机制

### 示例2：职业选择

**用户**：我在考虑要不要跳槽到一家创业公司

**选择视角**：
1. 企业家 - 创业公司的真实运作
2. 会计师 - 财务和风险分析
3. 心理学家 - 你真正想要什么
4. 棋手 - 这一步对未来几步的影响
5. 人类学家 - 两种公司文化的差异

### 示例3：产品问题

**用户**：用户留存率一直上不去

**选择视角**：
1. 医生 - 诊断留存低的"病因"
2. 销售员 - 用户真正想要什么
3. 设计师 - 产品如何与用户交流
4. 程序员 - 哪些可以自动化改善
5. 科学家 - 如何设计实验验证假设

---

## 6) 注意事项

1. **不是机械套用**：根据问题灵活选择，不是每次都用所有视角
2. **承认局限**：有些问题某些视角确实没用，直接说明
3. **避免过度发散**：最终要收敛到可执行的结论
4. **结合用户背景**：用户是交易员，就多用交易相关的类比
5. **可迭代**：用户可以要求增加某个视角或深入某个分析

---

## 7) 快捷指令

- `/think [问题]` - 自动选择视角，简洁模式分析
- `/think deep [问题]` - 深度模式，每个视角详细展开
- `/think as [职业] [问题]` - 指定用某个职业视角分析
- `/think vs [问题]` - 找出不同视角的矛盾和冲突
- `/think first [问题]` - 先做第一性原理拆解，再多视角分析
- `/think elon [问题]` - 纯第一性原理分析（马斯克模式）
