<action>
Process the validated draft and generate the final 4-file PRD:

### 1. Parse Validation Results

Read docs/prd/backfill-draft.md and categorize items:

| Status | Action |
|--------|--------|
| `[x]` INTENDED | Include in PRD as requirement |
| `[-]` BUG | Log to separate file, exclude from PRD |
| `[~]` DEPRECATED | Exclude from PRD, note for cleanup |
| `[?]` UNCLEAR | Flag for follow-up, exclude from PRD for now |

### 2. Transform to User Stories

For each INTENDED item, create a proper user story:

```
As a [user type]
I want to [action from observed behavior]
So that [inferred benefit]

Acceptance Criteria:
- [Derived from test assertions]
- [Derived from observed behavior]
```

### 3. Distribute Content to 4 Files

Split validated items into the appropriate PRD files:

**PRD.md (Sections 1-2, 11-14):**
- Section 1: Executive Summary (from overall system purpose)
- Section 2: Goals and Success Metrics (from business context)
- Section 11: MVP Scope (from validated features)
- Section 12-14: Risks, Timeline, Open Questions

**product-vision.md (Sections 3-5):**
- Section 3: User Personas (inferred from validated behaviors)
- Section 4: Epics and User Stories (from validated features)
- Section 5: User Journeys (from validated UI flows)

**requirements.md (Sections 6-7):**
- Section 6: Functional Requirements (from validated behaviors)
- Section 7: Non-Functional Requirements (from validated NFRs)

**technical-spec.md (Sections 8-10):**
- Section 8: Technical Architecture (from codebase analysis)
- Section 9: Data Model (from validated entities)
- Section 10: API Specifications (from validated routes)

### 4. Create Exclusion Report

Document items NOT included in PRD:
- Bugs to fix
- Deprecated code to remove
- Unclear items needing investigation

### 5. Generate Test Gap Analysis

Compare validated requirements to existing test coverage:
- Requirements with tests → ready for TDD
- Requirements without tests → need tests written
- Tests without requirements → dead tests to remove?

</action>
