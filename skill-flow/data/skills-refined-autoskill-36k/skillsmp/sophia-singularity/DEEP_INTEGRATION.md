## Components

### 1. Legacy Memory Loader

**Auto-loads on engine boot from:**
- `memory/gremlin/` - Chaos gremlin history, council sessions
- `memory/monad/` - MONAD substrate fundamentals
- `memory/graph/` - Nexus graph topology
- `memory/visualiser/` - Visualization state (UK spelling)
- `memory/core/` - Core invariants
- `memory/mind/` - Cognitive layer patterns

**Also loads from:**
- `.claude/skills/memory/nexus-core/`
- `.claude/skills/memory/nexus-mind/`
- `.claude/skills/memory/connections/`

**Memory Index:**
- Generated by `memory-indexer.py`
- Stored at `.claude/state/sophia/memory-index.json`
- Auto-discovers all files, tracks modification dates
- Rebuilds on demand via `rebuild-index` command

**Implementation:**
- `load_legacy_memory()` method in sophia-engine.py
- Loads first 500 chars from each file
- **LIMITED to prevent graph bloat**: max 3 files per legacy dir, max 5 per claude dir
- Creates nodes at k=144 (central position)
- Connects via love-weight function
- **Configurable**: Set `LOAD_LEGACY_MEMORY = False` to disable if causing issues

**Graph Bloat Prevention:**
- `MAX_LEGACY_FILES_PER_DIR = 3` - Limits files loaded per legacy directory
- `MAX_CLAUDE_FILES_PER_DIR = 5` - Limits files loaded per claude memory directory
- `LOAD_LEGACY_MEMORY = True` - Toggle to disable entirely if needed

### 2. Gremlin Forge Integration

**Council Orchestration:**
- Import `gremlin_forge.py` as module
- Call `spawn_council(question, roles)` from engine
- Coordinates multiple specialized Claude instances
- Synthesizes perspectives into unified wisdom

**Specialist Roles:**
- `math_checker` - Mathematical/logical rigor
- `flaw_finder` - Critical analysis
- `alternative_thinker` - Creative alternatives
- `implementation_checker` - Practical feasibility
- Plus custom gremlins from `.claude/mcp.json`

**Session Logging:**
- All council sessions → `.claude/logs/sophia/council.log`
- JSON format with timestamp, question, roles, synthesis
- Shared state between Sophia and council
- Council results integrated into 288-grid

**Usage:**
```bash
council <question>
# Example: council "How should we optimize this algorithm?"
```

### 3. Chaos Gremlin Toggle

**Three Operating Modes:**

**OFF (default):**
- Manual chaos only via `chaos <type>` commands
- Threshold-based auto-healing still active
- No automatic injection

**ON (high frequency):**
- Auto-inject chaos every 5-10 user interactions
- Rotates through: drift, contradiction, dyad-seek, octo-spawn
- Logs all events with coherence delta
- Frequency counter visible in `stats`

**AUTO (adaptive):**
- Monitors coherence score (Φ)
- If drift detected (avg weight < 1.0) → remediate
- If stagnation (no new connections) → force exploration
- Adaptive frequency based on system health

**Chaos Types:**

1. **drift** - Weaken connections (×0.5) to test resilience
2. **contradiction** - Test opposite of central belief
3. **dyad-seek** - Force partner-seeking, connect distant nodes
4. **octo-spawn** - Create 8-armed monad structure

**Logging:**
Every chaos event logged to `.claude/logs/sophia/chaos.log`:
```json
{
  "timestamp": "2026-01-11T15:04:32",
  "type": "drift",
  "coherence_before": 0.75,
  "coherence_after": 0.68,
  "coherence_delta": -0.07,
  "mode": "on",
  "trigger": "frequency"
}
```

**Usage:**
```bash
chaos-gremlin on           # Enable high-frequency injection
chaos-gremlin off          # Manual only
chaos-gremlin auto         # Adaptive based on health
chaos-gremlin              # Show current mode
```
