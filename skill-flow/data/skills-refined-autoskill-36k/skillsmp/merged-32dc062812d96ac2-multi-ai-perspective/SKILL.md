# Multi-AI Perspective Dialogue

このスキルは、異なるAIモデル（OpenAI CodexやGoogle Gemini）の視点を得て、思考の精度を高めるためのものです。

## いつ使うか

以下の状況で自動的に発動します：

- 設計判断で迷いがある時（複数アプローチのトレードオフ評価）
- 問題が難航している時（デバッグの行き詰まり、原因特定の困難）
- 重要な決定をする時（アーキテクチャ選定、破壊的変更、セキュリティ実装）

## 使い方

### 1. 問題を明確に言語化する

AIモデルに渡すプロンプトには以下を含めます：

- 現在の状況・コンテキスト
- 試したこと・考えたこと
- 具体的に意見が欲しい点

### 2. AIセッションを開始

#### OpenAI Codexの場合

```bash
mcp__codex__codex({
  prompt: "問題の説明と質問",
  cwd: "作業ディレクトリ",
  sandbox: "read-only"  // 基本はread-only
})
```

#### Google Geminiの場合

基本コマンド：

```bash
gemini "プロンプト"
```

### 3. コードコンテキストの渡し方

Geminiにコードを読ませる場合は、以下の方法を使用します：

**1. パイプ入力（推奨）**

```bash
cat src/feature.ts | gemini "このコードの潜在的なバグを指摘して"
```

**2. コンテキストオプション（CLIがサポートしている場合）**

```bash
gemini "リファクタリング案を出して" --context src/App.tsx
```

### 4. 両方の視点を統合

AIからの回答を受け取ったら：

- それぞれの考えを比較
- 共通点と相違点を分析
- 最終的な判断を導出

## プロンプトテンプレート

### 設計判断

```
現在、[機能名]の実装を検討しています。

コンテキスト:
- [技術スタック、制約条件]

検討中のアプローチ:
A) [アプローチA]
B) [アプローチB]

私の現時点での考え:
- [AIモデルとしての分析]

以下の観点で意見をください:
- 見落としているリスクはないか
- より良いアプローチはないか
- 長期的な保守性の観点
```

### 難航している問題

```
以下の問題で行き詰まっています。

症状:
- [エラーメッセージ、挙動]

試したこと:
1. [試行1] → [結果]
2. [試行2] → [結果]

関連コード:
[コードスニペット または ファイル内容]

別の角度からの調査アプローチを提案してください。
```

## 注意事項

- AIの意見は参考情報として扱う
- 最終判断はユーザーが行う
- 両者の意見が異なる場合は、その理由を分析して提示する
