# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Performance (perf)

**Impact:** CRITICAL
**Description:** JavaScript performance optimizations that prevent memory leaks, reduce DOM thrashing, and improve runtime efficiency.

## 2. Async Patterns (async)

**Impact:** CRITICAL
**Description:** Proper handling of asynchronous operations, promises, and async/await patterns to prevent race conditions and improve responsiveness.

## 3. Security (security)

**Impact:** HIGH
**Description:** Security best practices to prevent XSS, injection attacks, and other common JavaScript vulnerabilities.

## 4. Error Handling (error)

**Impact:** HIGH
**Description:** Robust error handling patterns for catching, logging, and recovering from runtime errors gracefully.

## 5. ES Modules (module)

**Impact:** MEDIUM-HIGH
**Description:** Proper use of ES modules for tree-shaking, code splitting, and maintainable code organization.

## 6. Data Structures (data)

**Impact:** MEDIUM
**Description:** Choosing appropriate data structures (Map, Set, Array) for optimal performance and code clarity.

## 7. DOM Manipulation (dom)

**Impact:** MEDIUM
**Description:** Efficient DOM manipulation patterns that minimize reflows and repaints.

## 8. Modern Features (modern)

**Impact:** LOW-MEDIUM
**Description:** Leveraging modern JavaScript features (ES2020+) for cleaner, more maintainable code.
