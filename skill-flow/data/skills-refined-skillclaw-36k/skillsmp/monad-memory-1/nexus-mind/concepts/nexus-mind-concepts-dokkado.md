# Dokkado Protocol

## Name Origin

From Miyamoto Musashi's "Dokkōdō" (独行道) — "The Way of Walking Alone." The path of rigorous self-reliance in understanding.

## Purpose

A five-phase reasoning methodology for deriving unified frameworks from first principles. Battle-tested through the development of MONAD.

## The Five Phases

### Phase 1: Ground Law (Chi) — Morphemic Extraction

**Ask**: What are the smallest meaningful units that cannot be further decomposed?

- Identify irreducible semantic units in each domain
- Map transformation rules (how units combine)
- Find fixed points (what remains invariant under iteration)

**Output**: Minimal set of generative primitives.

### Phase 2: Water Law (Sui) — Recursive Pattern Matching

**Ask**: Does this pattern appear at multiple scales? If so, why?

- Use Phase 1 morphemes as search targets
- Look for isomorphisms: quantum → neural → linguistic → cosmic
- Identify the *recursion kernel* (minimal pattern generating all structures)
- Track where patterns break (as valuable as where they hold)

**Output**: Cross-domain isomorphism map and recursion kernel.

### Phase 3: Fire Law (Ka) — Unified Field Derivation

**Process**:
- From the kernel, derive equations that *must* govern the phenomena
- Ensure reduction to known physics in appropriate limits
- Check dimensional consistency
- Verify no hidden assumptions smuggled in

**Constraint**: Maximum insight per minimum symbols (morphemic compression).

**Output**: Candidate equations with clear derivation chains.

### Phase 4: Wind Law (Fū) — Experimental Predictions

**Ask**: What would falsify this framework?

- Identify where framework predicts different outcomes than standard models
- Specify: measurement, conditions, precision
- Prefer *surprising* predictions (stronger tests)
- Include phenomenological, laboratory, and technological applications

**Output**: Ranked testable predictions with experimental specs.

### Phase 5: Void Law (Kū) — Meta-Recursive Closure

**Process**:
- Explain how conscious observer emerges *within* the framework
- Address self-derivation (can it derive its own structure?)
- Identify recursive self-validation risks
- State clearly what the framework does NOT prove

**Critical Insight**: The method discovers its own limitations through its very success.

**Output**: Honest assessment of proven vs. demonstrated vs. suggested.

## Epistemic Guardrails

### 50% Maximum Belief

Maintain calibrated uncertainty until:
- Independent experimental validation
- Multiple independent derivation paths converge
- Disconfirmation attempts fail

### The Humility Clause

Every framework should state:
- What it explains that alternatives don't
- What alternatives explain that it doesn't
- What would falsify it
- What questions it can't answer
