## 📋 清理层级速查表

### 层级2：RFC完成清理

| 项目 | 说明 |
|------|------|
| **触发条件** | RFC audit通过，状态变为completed |
| **执行时机** | 立即（开始新RFC前） |
| **优先级** | 高 |
| **主要动作** | 生成摘要、压缩日志、清理临时文件、更新索引 |
| **预期效果** | 减少50-70%的RFC文件大小 |
| **耗时** | 5-10分钟 |
| **关键文件** | progress.json、findings.json、rfc-index.md |
| **检查清单** | 5步（见下表） |

**5步检查清单**：
```
□ 步骤1：生成RFC摘要 → .claude/archive/rfc-[ID]-[date].md
□ 步骤2：压缩progress.json → 删除interactions、decisions详情
□ 步骤3：压缩findings.json → 删除code_snippets
□ 步骤4：清理临时文件 → 删除*.tmp、precheck-report-*.md
□ 步骤5：更新索引 → rfc-index.md、tech-stack-index.md等
```

## 🔍 故障排查速查表

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 清理后Token未下降 | 清理不完全 | 检查清理日志，重新执行 |
| 无法找到RFC | 索引未更新 | 查看rfc-index.md，搜索archive/ |
| 清理过度丢失信息 | 策略过激进 | 从检查点恢复 |
| 清理脚本失败 | 权限或路径错误 | 检查权限、路径、jq安装 |
| 检查点文件损坏 | 写入中断 | 使用最近的备份检查点 |
| Token计算不准 | 估算方法差异 | 与系统报告对比，调查差异 |

## ⚡ 快速操作命令

### 查看当前状态
```bash
# 查看Token使用率
cat .claude/ohspec/rfcs/*/progress.json | jq '.token_usage'

# 查看RFC列表
cat .claude/rfc-index.md

# 查看活跃RFC
ls -la .claude/ohspec/rfcs/

# 查看归档统计
ls -lh .claude/archive/ | tail -20
```

### 执行清理
```bash
# 执行层级2清理
bash cleanup-level2.sh RFC-20260116-xxx

# 执行黄色清理
bash cleanup-yellow.sh

# 检查Token使用率
bash check-token-usage.sh
```

### 恢复操作
```bash
# 查看可用检查点
ls -lt .claude/checkpoint-*.json | head -5

# 查看紧急检查点
ls -lt .claude/emergency-checkpoint-*.json | head -1

# 从检查点恢复（在新会话中）
/ohspec:resume RFC-20260116-xxx
```

## 📞 决策速查

### 何时执行清理？

**立即执行**：
- ✅ RFC刚完成（audit通过）
- ✅ Token达到50%
- ✅ Token达到70%
- ✅ Token达到85%

**计划执行**：
- 📅 完成5个RFC后（层级4）
- 📅 每周日（层级7）
- 📅 项目里程碑（层级4+9）

### 清理优先级

**高优先级**（立即执行）：
1. 红色清理（Token 85%）
2. RFC完成清理（层级2）
3. 黄色清理（Token 50%）

**中优先级**（计划执行）：
4. 橙色清理（Token 70%）
5. 知识库提炼（层级4）
6. 定期维护（层级7）

**低优先级**（可延后）：
7. 里程碑清理（层级4+9）

### 保留vs删除

**必须保留**：
- ✓ 当前活跃RFC的完整上下文
- ✓ RFC摘要和索引
- ✓ 关键决策和风险
- ✓ 检查点文件

**可以删除**：
- ✗ 代码片段（保留文件路径）
- ✗ 详细的交互记录
- ✗ 临时分析文件
- ✗ 过期的precheck报告

**应该归档**：
- 📦 已完成的RFC
- 📦 历史日志
- 📦 过期的检查点

## 🎯 最佳实践速记

### DO（推荐）

✅ **定期检查Token使用**
- 每开始新RFC前检查
- 达到50%时主动清理

✅ **及时更新索引**
- 每次清理后更新
- 保持索引最新

✅ **创建检查点**
- 重要RFC开始前
- 里程碑时创建

✅ **保持日志精简**
- 及时压缩progress.json
- 及时压缩findings.json

✅ **定期提炼知识库**
- 不要等到RFC太多
- 及时提炼提高复用率

### DON'T（避免）

❌ **不要过度清理**
- 不要删除当前RFC上下文
- 不要清理高相关性历史RFC

❌ **不要忽略索引**
- 清理后必须更新索引
- 没有索引的归档很难恢复

❌ **不要跳过检查点**
- 重大清理前必须创建
- 检查点是最后保险

❌ **不要手动删除归档**
- 使用清理策略自动管理
- 手动删除可能破坏索引

❌ **不要忽视Token监控**
- 定期检查使用情况
- 不要等到红色警告
