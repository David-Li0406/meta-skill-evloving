# Clarifying Questions by Requirement Type

This document provides question templates organized by requirement type and missing criteria.

## Questions by Missing Criterion

### Missing Unambiguous Formulation
1. "Can you provide specific measurable criteria for [vague term]?"
2. "What exactly does [term] mean in this context?"
3. "What are the acceptable ranges/values for [parameter]?"

### Missing Completeness
**Inputs**: "What data will be provided as input? What format?"
**Outputs**: "What should the system produce? How are results displayed?"
**Constraints**: "Are there performance, security, or compliance requirements?"

### Missing Verifiability
1. "How will we know this requirement is met?"
2. "What test scenario would verify this?"
3. "What are the specific acceptance criteria?"

### Missing Rationale
1. "What problem does this solve?"
2. "What is the business value?"
3. "What user need drives this requirement?"

## Questions by Requirement Type

### Feature Requests
1. "What specific user problem does this feature solve?"
2. "Who are the target users?"
3. "What are the key use cases?"
4. "What are the success metrics?"

### Bug Reports
1. "What steps reproduce the bug?"
2. "What is the expected vs actual behavior?"
3. "What environment/conditions trigger this?"
4. "Is this consistently reproducible?"

### API Changes
1. "What is the HTTP method and endpoint path?"
2. "What are the request/response schemas?"
3. "What are the error codes and scenarios?"
4. "Is authentication required?"

### Database Changes
1. "What tables/collections are affected?"
2. "What is the schema change?"
3. "Is a migration required for existing data?"
4. "Are there performance implications?"
