# Skill body

## Capabilities
- Generating API documentation
- Creating developer guides
- Documenting architecture
- Creating user manuals
- Generating OpenAPI/Swagger specs
- Updating existing documentation

## Instructions

### Execution Process

#### Step 1: Identify Documentation Type
Determine the type of documentation needed:
- **API Documentation**: Endpoint references
- **Developer Guide**: Setup and usage instructions
- **Architecture Docs**: System overview
- **User Manual**: Feature guides

#### Step 2: Extract Information
Gather content for documentation:
- Read code and comments
- Analyze API endpoints
- Extract examples
- Understand system architecture

#### Step 3: Generate Documentation
Create the documentation using the following guidelines:
- Follow the appropriate documentation templates
- Include comprehensive examples
- Add troubleshooting sections
- Ensure a clear and logical structure

#### Step 4: Validate Documentation
Check the quality of the generated documentation:
- Verify completeness
- Ensure examples are functional
- Check for clarity and coherence
- Validate all links

## Best Practices
1. **Extract from Code**: Use code as the primary source of truth for documentation.
2. **Include Examples**: Provide working examples to enhance understanding.
3. **Keep Updated**: Regularly sync documentation with code changes.
4. **Clear Structure**: Organize documentation logically for ease of navigation.
5. **User-Focused**: Write documentation with the end-user in mind, not just for the system.

## Integration
**Integration with Technical Writer Agent**:
- Utilizes this skill for generating and validating documentation quality.

**Integration with Developer Agent**:
- Generates API documentation and creates inline documentation.
- Updates documentation in response to code changes.
