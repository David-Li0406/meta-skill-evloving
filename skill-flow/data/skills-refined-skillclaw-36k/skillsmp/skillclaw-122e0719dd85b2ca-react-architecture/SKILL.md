# React Architecture Guidelines

This skill provides guidelines and best practices for structuring and organizing React and React Native applications.

## Steps

1. **Define Project Structure**: Organize your project into clear directories for components, assets, and utilities.
2. **Component Design**: Create reusable components that follow the single responsibility principle.
3. **State Management**: Choose an appropriate state management solution (e.g., Context API, Redux) based on the complexity of your application.
4. **Routing**: Implement routing using libraries like React Router for web applications or React Navigation for mobile applications.
5. **Styling**: Decide on a styling approach (CSS Modules, styled-components, etc.) that fits your team's workflow.
6. **Testing**: Write unit and integration tests for your components to ensure reliability.
7. **Documentation**: Maintain clear documentation for your architecture decisions and component usage.

## Best Practices

- Keep components small and focused.
- Use prop types or TypeScript for type checking.
- Regularly review and refactor your code to improve maintainability.
