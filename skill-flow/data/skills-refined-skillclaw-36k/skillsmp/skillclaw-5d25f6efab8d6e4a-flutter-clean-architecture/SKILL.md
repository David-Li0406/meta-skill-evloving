# Flutter Clean Architecture Guidelines

This skill provides guidelines and best practices for implementing Clean Architecture in Flutter applications. Follow these steps to ensure a well-structured and maintainable codebase.

1. **Define Layers**: Organize your application into layers (Presentation, Domain, Data) to separate concerns.
2. **Use Dependency Injection**: Implement dependency injection to manage dependencies between layers effectively.
3. **Create Use Cases**: Define use cases in the Domain layer to encapsulate business logic.
4. **Implement Repositories**: Use repositories in the Data layer to abstract data sources and provide a clean API for the Domain layer.
5. **Follow SOLID Principles**: Adhere to SOLID principles to enhance code maintainability and scalability.
6. **Write Tests**: Ensure that each layer is thoroughly tested, focusing on unit tests for business logic and integration tests for data sources.

For more detailed documentation, refer to the official guidelines.
