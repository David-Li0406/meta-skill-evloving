# 技术信息提取指南

## 需要提取的关键信息

### 1. 产品标识信息
优先从邮件中提取以下信息：

- **产品型号**: MCTRL4K、LED屏型号等
- **序列号 (S/N)**: 通常格式为 xxxxxxxxxxxxxxxx
- **固件版本**: 如 1.3.00、v2.1等
- **硬件版本**: 如有的话
- **相关配件**: 接收卡、发送卡、线缆类型等

### 2. 故障现象描述
提取客户报告的具体症状：

- **指示灯状态**: 闪烁频率、颜色
- **显示问题**: 无显示、花屏、闪烁等
- **信号识别**: 能否识别输入信号
- **错误提示**: 任何错误消息或代码
- **异常行为**: 与正常情况的差异

**注意**: 区分症状和原因，只记录观察到的现象

### 3. 排查步骤和结果
提取已完成的测试：

- ✅ 正常工作的功能
- ❌ 异常的功能
- 🔄 未测试的项目

常见排查项目：
- 不同信号源测试（PC、笔记本等）
- 不同分辨率测试
- 不同线缆测试（HDMI、DP等）
- 软件测试（如NovaLCT）
- 硬件互换测试

### 4. 环境和背景信息
提取可能相关的背景：

- **使用场景**: 演出、展览、日常使用等
- **异常事件**: 电源浪涌、摔落、进水等
- **时间信息**: 故障发生时间、持续时间
- **环境因素**: 温度、湿度、电源稳定性
- **近期变更**: 升级、移动、安装等

### 5. 行动事项
提取明确的待办事项：

- **需要研发**: 分析日志、提供解决方案、确认硬件故障等
- **需要技术支持**: 指导客户、收集更多信息等
- **需要客户**: 测试、提供照片/视频、返修等

### 6. 参与人员
识别邮件中的关键人物：

**人员识别规则：**
- **技术支持人员**: 邮箱域名为 @novastar.tech 的都是公司内部人员
  - 特殊识别: zhangyw1@novastar.tech 是 technical support engineer（本人）
  - 其他 @novastar.tech 域名: 技术支持、研发、销售等内部团队
- **客户**: 所有非 @novastar.tech 域名的人员

**角色分类：**
- **客户**: 报告问题的人，公司、职位（非 novastar.tech 域名）
- **技术支持**: 处理案例的工程师（novastar.tech 域名）
- **销售/商务**: 抄送的商务人员（novastar.tech 域名）
- **其他相关方**: 如第三方、服务商等（非 novastar.tech 域名）

## 提取技巧

### 识别关键信息
- 序列号通常是字母数字组合，10-20位
- 固件版本常为 x.x.x 格式
- 技术术语通常全大写或驼峰命名

### 过滤噪音
忽略以下内容：
- 客套话（"Thank you", "Best regards"等）
- 签名档（公司信息、地址等）
- 节假日通知
- 重复引用的历史邮件（除非有新信息）

### 邮件线程处理
对于多轮邮件：
- 按时间顺序提取信息
- 最新的回复通常包含最新状态
- 合并同类的技术信息
- 更新已完成的排查项

### 优先级
信息提取优先级：
1. **必须**: 序列号、产品型号、核心故障现象
2. **重要**: 固件版本、已尝试的排查、环境背景
3. **参考**: 联系方式、公司信息、历史案例
