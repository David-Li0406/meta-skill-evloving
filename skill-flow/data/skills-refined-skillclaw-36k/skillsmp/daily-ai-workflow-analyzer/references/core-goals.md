# 核心目标和价值

> 文档版本：v1.0
> 创建时间：2026-01-12
> 适用范围：daily-ai-workflow-analyzer 技能

---

## 🎯 核心定位

**不是**：数据统计分析工具
**而是**：知识生成与价值提取系统

### 价值主张

从语音聊天记录中提取**可复用的知识资产**，帮助用户：
- 发现潜在问题
- 形成知识沉淀
- 提供解决方案

---

## 📦 三大输出维度

### 1. 沟通中的问题

**目标**：提升沟通效率和质量

#### (a) 分析用户与AI在对话中的表现
- 指令清晰度：是否包含目标/约束/期望
- 上下文连续性：是否需要频繁澄清
- 情绪影响度：情绪是否影响表达清晰度
- 迭代效率：达成共识需要的轮次

#### (b) 评估编程沟通等方面的改善空间
- 技术问题识别
- 需求表达能力
- 协作模式优化

**输出形式**：
```
问题类型 | 表现 | 根因 | 解决方案 | 预期效果
---------|------|------|----------|----------
指令模糊 | "帮我分析一下" | 缺少上下文 | 增加背景说明 | 减少来回对话
重复提问 | 同样的问题问了3次 | 未形成记忆 | 知识库同步 | 效率提升50%
```

---

### 2. 结构化建议

**目标**：优化现有流程和工作方式

#### (a) 针对现有流程的优化建议
- 工作流瓶颈识别
- 自动化机会发现
- 流程简化方案

#### (b) 针对SOP（标准作业程序）的建议
- 发布前检查清单
- 测试验证流程
- 需求管理规范

**输出形式**：
```
维度 | 当前状态 | 问题识别 | 优化建议 | 优先级
------|---------|---------|---------|-------
工作流 | 手动同步进展 | 容易遗漏 | AI自动同步 | P0
测试流程 | 依赖人工发现 | P0问题漏检 | 自动化测试 | P0
需求管理 | 分散在对话中 | 难以追溯 | 集中式需求追踪 | P1
```

---

### 3. 沟通工具的生成

**目标**：提供可复用的沟通模板和工具

#### (a) 帮我生成一些提示词（Prompt）
- 编程任务模板
- 问题反馈模板
- 知识沉淀模板

#### (b) 指导我应该如何更好地告诉AI去执行任务，以及如何优化沟通方式
- 沟通指南
- 最佳实践
- 常见问题FAQ

**输出形式**：
```markdown
## 高效Prompt模板库

### 编程任务模板
背景：{项目背景}
目标：{具体目标}
约束：{技术约束}
期望：{输出形式}

### 问题反馈模板
现象：{具体现象}
频率：{复现频率}
影响：{影响范围}
优先级：{P0/P1/P2}
```

## 🔍 核心价值流

```
输入：分场景的语音聊天记录（按APP/工作流维度）
  ↓
处理：AI + 程序协作（智能预处理 + 深度分析）
  ↓
输出：
  1. 问题诊断卡片（发现问题 + 根因 + 解决方案 + 预期效果）
  2. 流程优化SOP（背景 + 目标 + 步骤 + 验证 + 注意事项）
  3. 沟通工具库（Prompt模板 + 沟通指南 + 常见问题FAQ）
  ↓
验证：效果追踪 + 反馈闭环 + 知识更新
```

---

## 💡 设计原则

### 1. 价值优先
- 不做无意义的统计
- 聚焦可行动的洞察
- 每个输出都要有明确的应用场景

### 2. 结构化输出
- 使用标准化格式
- 便于机器理解和复用
- 适合知识库沉淀

### 3. 闭环优化
- 提供验证方法
- 收集反馈
- 持续迭代改进

### 4. Token效率
- 程序预处理减少AI处理量
- 聚焦高价值记录
- 用便宜的模型做初步分析
