# Skill body

1. **Prepare for Migration**
   - Review the current state of your Agentic QE project in version 2.
   - Backup all project data to ensure no data is lost during the migration process.

2. **Set Up the New Environment**
   - Install the necessary tools and dependencies for version 3.
   - Ensure that the new environment is configured correctly to support the migration.

3. **Migrate Data**
   - Use the migration tool provided for Agentic QE to transfer data from version 2 to version 3.
   - Monitor the migration process for any errors or issues.

4. **Verify Data Integrity**
   - After migration, check that all data has been transferred correctly.
   - Run tests to ensure that the project functions as expected in version 3.

5. **Finalize Migration**
   - Once verification is complete, finalize the migration by cleaning up any temporary files or configurations.
   - Document the migration process and any changes made for future reference.
