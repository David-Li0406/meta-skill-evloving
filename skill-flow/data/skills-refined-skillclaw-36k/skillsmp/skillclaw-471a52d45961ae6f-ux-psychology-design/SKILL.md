# UX Psychology and Design

This skill applies principles of UX psychology to enhance UI design, making it more user-friendly and effective.

## Usage

```
/ux-psychology-design <file or URL or screenshot>
```

Analyze the design using psychological principles and provide improvement suggestions.

## Key Psychological Principles

### Cognitive and Attention Principles

| Principle | Description | Application Example |
|-----------|-------------|---------------------|
| Cognitive Load | The mental effort required to process information | Present information in stages, break complex tasks into smaller steps |
| Selective Attention | The tendency to focus on relevant information | Visually emphasize important elements |
| Decision Fatigue | Diminished ability to make decisions after a long session of decision-making | Reduce choices, set default values |

### Memory and Learning Principles

| Principle | Description | Application Example |
|-----------|-------------|---------------------|
| Serial Position Effect | Items at the beginning and end are more easily remembered | Place important items at the start or end |
| Zeigarnik Effect | Unfinished tasks are more memorable | Use progress bars, visualize incomplete tasks |

### Bias and Judgment Principles

| Principle | Description | Application Example |
|-----------|-------------|---------------------|
| Anchoring Effect | The first piece of information serves as a reference point | Display high-priced plans first |
| Loss Aversion | The tendency to prefer avoiding losses over acquiring equivalent gains | Use phrases like "Don't miss out" |

### Motivation and Behavior Principles

| Principle | Description | Application Example |
|-----------|-------------|---------------------|
| Goal Gradient Effect | Motivation increases as one gets closer to a goal | Display progress bars, show remaining steps |
| Gradual Commitment | Starting with small requests leads to larger commitments | Begin with easy actions |

### Social and Trust Principles

| Principle | Description | Application Example |
|-----------|-------------|---------------------|
| Social Proof | People are influenced by the actions and opinions of others | Show reviews, user counts, "popular" labels |
| Authority Bias | Trust in the opinions of experts | Display media mentions, qualifications |

## Application Patterns

### Landing Pages

| Section | Applied Effect | Implementation Example |
|---------|----------------|------------------------|
| Hero Section | Framing Effect | Emphasize positive outcomes |
| CTA | Loss Aversion | "Don't miss this opportunity" |
| Pricing Table | Anchoring Effect | Show high-priced plans first |

### Onboarding

| Step | Effect | Implementation Example |
|------|--------|------------------------|
| Initial | Progressive Disclosure | One screen, one concept |
| Ongoing | Goal Gradient Effect | Show progress bar |
| Completion | Peak-End Rule | Celebration animation |

## Improvement Suggestions

### Current Issues and Recommendations

| Principle | Current Issue | Improvement Suggestion | Expected Effect |
|-----------|---------------|------------------------|------------------|
| Cognitive Load | Too much information at once | Break down information into smaller chunks | Easier decision-making |
| Loss Aversion | Weak CTAs | Use stronger language in CTAs | Increased conversion rates |

## Important Notes

- Avoid excessive application of principles that could lead to dark patterns.
- Always prioritize user benefit.
- Conduct A/B testing to validate effectiveness.

## Checklist for Application

- [ ] Minimize cognitive load (one action per screen)
- [ ] Ensure aesthetic usability
- [ ] Appropriately place social proof
- [ ] Apply loss aversion in CTAs
- [ ] Display progress to leverage goal gradient effect
- [ ] Confirm elimination of dark patterns
