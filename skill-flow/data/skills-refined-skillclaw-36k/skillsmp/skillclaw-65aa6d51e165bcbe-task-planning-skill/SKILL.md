# Task Planning Skill

This skill helps users break down projects into manageable tasks and ensures that all work is reviewed for quality and completeness.

## When to Use This Skill

Use this skill when:
- You have a large project that needs to be divided into smaller tasks.
- You want to ensure that all tasks are completed and reviewed.
- You need to track progress and identify areas for improvement.

## Core Principles

### 1. Break Down Tasks

Divide your project into smaller, actionable tasks to make it more manageable.

**Example:**
```yaml
# Example of task breakdown
tasks:
  - task: Research topic
  - task: Create outline
  - task: Write draft
```

### 2. Review Process

Always include a final review task to assess the completed work and identify any necessary fixes or enhancements.

```yaml
# Example of a review task
review:
  task: Review completed work
```

## Common Patterns

### Pattern 1: Task List

```yaml
# Show a complete task list example
task_list:
  - task: Initial research
  - task: Drafting sections
  - task: Final review
```

**Explanation:** This pattern helps in organizing tasks in a sequential manner.

### Pattern 2: Review Checklist

Provide a checklist to ensure all aspects of the work are reviewed.

```yaml
# Example of a review checklist
review_checklist:
  - Check for completeness
  - Verify accuracy
  - Ensure clarity
```

## Best Practices

1. **Plan Thoroughly** - Always plan your tasks before starting.
2. **Review Regularly** - Incorporate regular reviews into your workflow.
3. **Adjust as Needed** - Be flexible and adjust tasks based on progress and feedback.

## Common Issues and Solutions

### Issue: Overwhelmed by Tasks

**Symptoms:**
- Feeling lost or unsure where to start.
- Tasks seem too large or unmanageable.

**Solutions:**
1. Break tasks into smaller, more manageable pieces.
2. Prioritize tasks based on urgency and importance.
3. Set specific deadlines for each task.

**Example:**
```yaml
# Example of a prioritized task list
prioritized_tasks:
  - task: Urgent research
  - task: Important drafting
```
