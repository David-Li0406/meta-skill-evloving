# Architecture Guidelines for React and React Native

This skill provides guidelines and best practices for structuring applications built with React and React Native. 

1. **Define Project Structure**: Organize your project into clear directories for components, assets, and services.
2. **Component Design**: Create reusable components that follow the single responsibility principle.
3. **State Management**: Choose an appropriate state management solution (e.g., Context API, Redux) based on the complexity of your application.
4. **Routing**: Implement routing using libraries like React Router for web or React Navigation for mobile.
5. **Styling**: Use consistent styling practices, such as CSS Modules or styled-components, to maintain a clean UI.
6. **Testing**: Write unit and integration tests for your components to ensure reliability.
7. **Documentation**: Maintain clear documentation for your architecture decisions and component usage.

For detailed documentation, refer to the architecture guidelines in your project.
