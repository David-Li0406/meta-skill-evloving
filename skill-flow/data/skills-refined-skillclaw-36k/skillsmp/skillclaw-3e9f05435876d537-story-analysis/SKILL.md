# Skill body

## 功能

综合分析故事的题材、情节和人物关系，识别关键元素与发展变化，提供深度理解和优化建议。

## 使用场景

- 识别故事的核心题材和类型定位，明确市场方向。
- 深度分析故事情节结构，揭示内在逻辑。
- 理解人物关系网络，分析其对情节的推动作用。
- 为故事改编和商业化提供深度参考依据。

## 核心步骤

1. **题材分析**:
   - 识别故事的主要题材类型和次要题材元素。
   - 提炼故事的独特创意元素，增强作品吸引力。

2. **情节点分析**:
   - 识别故事中的所有重要情节点。
   - 分析情节点的重要性级别（核心/重要/辅助）及其对整体故事的影响。

3. **人物关系分析**:
   - 识别故事中的所有主要人物及其关系类型。
   - 描述人物关系的具体特点和发展过程。

## 输入要求

- **故事文本/大纲/梗概**: 完整的原始故事文本、大纲或梗概。
- **文本长度建议**: 500字以上。

## 输出格式

```
【题材类型分析】
- 主导题材：[题材类型]
- 辅助题材：[题材类型1、题材类型2]
- 题材融合度：[高/中/低]

【情节点分析】
- 情节点 X：情节点名称
  - 类型：[开端/发展/转折/高潮/结局]
  - 重要性级别：[核心/重要/辅助]
  - 影响范围：描述对整体故事的影响

【人物关系分析】
- 主要人物列表：[人物列表]
- 关系类型统计：[各类关系数量]
```

## 约束条件

- 分析结果需忠实于原始故事文本，不进行主观臆断。
- 确保题材、情节点和人物关系的分类准确，描述清晰。
- 建议具备可操作性，能指导情节优化和人物塑造。
