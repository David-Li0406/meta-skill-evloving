# Skill body

## Overview

The Daily Workflow framework is designed to facilitate the management of tasks throughout the day, ensuring that work is received, processed, and logged effectively. It complements the ICE (single-task execution) framework by providing a comprehensive workspace for multiple tasks.

## Core Principles

- **ICE** is a single-task closed loop (Intent → Condition → Eval).
- **Daily Workflow** is a cross-task workspace (in → out → log + artifacts).
- At the end of each day, clear the in/out sections, log important items, and store artifacts for future reference.

## Structure

### Files and Directories

| File/Directory | Action | Lifecycle | Core Question |
|:---:|:---:|:--------:|----------|
| **in.md** | Receive | Daily | What did I receive today? |
| **out.md** | Send | Daily | What did I produce today? |
| **log.md** | Store | Permanent | What is worth keeping? |
| **artifacts/** | Deliver | Permanent | What are the lasting outputs? |

## Workflow Steps

1. **Receive Tasks**: Use `in.md` to collect ideas, tasks, and notes without judgment or categorization.
2. **Process Tasks**: Execute each task using the ICE framework.
3. **Send Outputs**: Use `out.md` to document results and distribute them to the appropriate channels.
4. **Log Important Items**: At the end of the day, review the `in.md` and `out.md` to log significant items in `log.md`.
5. **Store Artifacts**: Save any important outputs in the `artifacts/` directory for future reference.

## Daily Routine

- At the start of the day, review the `in.md` for tasks to be processed.
- Throughout the day, update `out.md` with completed tasks and results.
- At the end of the day, clear `in.md` and `out.md`, log important items, and ensure artifacts are stored properly.

By following this structured approach, you can maintain a clear overview of your daily tasks and ensure that important work is preserved for future use.
