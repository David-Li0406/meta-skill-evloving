# Coding Standards for React and React Native with TypeScript

This skill provides guidelines and best practices for maintaining consistent coding standards in React and React Native projects that utilize TypeScript.

## Guidelines

1. **File Naming Conventions**: Use PascalCase for component files and camelCase for utility files.
2. **Component Structure**: Each component should be in its own folder, containing the component file and its associated styles.
3. **Type Definitions**: Always define prop types using TypeScript interfaces or types.
4. **State Management**: Use hooks for managing state in functional components.
5. **Styling**: Prefer styled-components or CSS modules for styling components.
6. **Linting**: Use ESLint with a TypeScript configuration to enforce coding standards.
7. **Documentation**: Document components and functions using JSDoc comments.

For detailed documentation, refer to the coding standards documentation.
