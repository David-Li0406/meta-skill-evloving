# 选题策划 Skills 系统

完整的内容产品（图书/课程/专栏/IP）选题策划工具集，帮助您从模糊想法到完整策划案。

## 🎯 系统概览

这是一套模块化的选题策划 Skills，采用**关注点分离原则**，每个 Skill 专注一个核心任务。

### 适用场景
- 📚 图书选题策划
- 🎓 在线课程设计
- 📰 专栏/训练营规划
- 🎭 IP内容延展开发

## 📦 Skill 清单

### 第一阶段：核心策划流程（6个Skills）

#### 1. topic-planning（主入口）
**功能**：识别策划场景，引导进入合适的 Skill 流程

**使用场景**：
- 刚开始策划，不确定从哪里入手
- 需要了解完整的策划流程
- 想快速找到适合自己的路径

**输出**：场景识别 + 推荐路径 + 启动下一个 Skill

---

#### 2. topic-scene-guide（场景分析）
**功能**：识别业务场景、干系人、约束条件

**使用场景**：
- 新立项，需要明确项目背景
- 涉及多方干系人，需要梳理诉求
- 有复杂的约束条件需要分析

**输出**：场景分析报告（干系人地图、约束条件、策划重点）

---

#### 3. topic-research（市场调研）⭐ 增强版
**功能**：通过联网搜索和交叉验证，获取可信的市场情报

**使用场景**：
- 需要了解竞品情况
- 需要市场数据支持决策
- 需要验证市场机会

**核心特性**：
- ✅ 交叉验证机制（3+来源验证）
- ✅ 可信度评级系统（5星评级）
- ✅ 详细来源标注（标题、时间、渠道、链接）
- ✅ 不可信信息过滤

**输出**：市场调研报告（竞品分析、用户洞察、市场机会）

---

#### 4. topic-positioning（定位设计）⭐ 核心环节
**功能**：提炼价值主张和差异化策略

**使用场景**：
- 需要明确"为谁解决什么问题"
- 需要找到差异化定位
- 需要提炼核心卖点

**重要性**：这是整个策划的核心环节，定位决定成败

**输出**：定位文档（用户画像、价值主张、差异化策略、内容策略）

---

#### 5. topic-structure（结构设计）
**功能**：规划章节框架、学习路径、内容编排

**使用场景**：
- 定位确定后，需要设计内容结构
- 需要规划学习路径
- 需要评估工作量

**输出**：结构设计文档（章节框架、学习路径、详细大纲、可行性评估）

---

#### 6. topic-proposal（策划案撰写）
**功能**：整合所有成果，输出完整的 Markdown 策划文档

**使用场景**：
- 前期工作完成，需要形成正式文档
- 需要用于立项决策或对外沟通

**输出**：完整策划案（Markdown格式）

---

### 第二阶段：质量保障（1个Skill）

#### 7. topic-review（质量检查）
**功能**：对已有策划案进行全面评审，提供优化建议

**使用场景**：
- 策划案完成后，需要质量检查
- 已有方案，需要第三方评审
- 发现问题，需要针对性优化

**三种评审模式**：
- 快速诊断（15分钟）
- 全面评审（1小时）
- 专项检查（30分钟）

**输出**：评审报告（问题诊断、优化建议、风险清单）

---

## 🚀 使用指南

### 推荐流程

#### 场景A：新立项（从0到1）
```
topic-planning（入口）
    ↓
topic-scene-guide（明确场景）
    ↓
topic-research（市场调研）
    ↓
topic-positioning（核心定位）⭐
    ↓
topic-structure（结构设计）
    ↓
topic-proposal（撰写策划案）
    ↓
topic-review（质量检查）
```

#### 场景B：IP延展（从1到N）
```
topic-planning（入口）
    ↓
topic-scene-guide（明确场景）
    ↓
topic-positioning（重新定位）⭐
    ↓
topic-structure（结构设计）
    ↓
topic-proposal（撰写策划案）
```

#### 场景C：优化迭代（已有初稿）
```
topic-planning（入口）
    ↓
topic-review（质量检查）
    ↓
根据评审结果，针对性调用相应Skill优化
```

### 灵活使用

✅ **支持跳过步骤**：如已有市场调研，可直接进入定位设计  
✅ **支持非线性**：可随时返回修改某个环节  
✅ **支持单点使用**：每个Skill都可独立使用

## ⭐ 核心特性

### 1. 关注点分离
每个 Skill 只做一件事，边界清晰，易于使用和维护。

### 2. 混合交互模式
- 对话式：自然语言交流，灵活高效
- 表单式：结构化输入，确保完整性

### 3. 增强的信息验证机制
`topic-research` 内置完整的信息搜索与验证体系：
- **交叉验证**：关键信息必须3+来源验证
- **可信度评级**：5星评级系统，过滤不可信信息
- **来源标注**：每条信息标注标题、时间、渠道、链接

### 4. Markdown 输出
所有成果输出为标准 Markdown 格式，易于编辑和分享。

## 📋 输出成果

使用完整流程后，您将获得：

1. **场景分析报告**：明确干系人、约束、策划重点
2. **市场调研报告**：竞品分析、用户洞察、市场机会（附详细来源）
3. **定位文档**：用户画像、价值主张、差异化策略
4. **结构设计文档**：章节框架、学习路径、详细大纲
5. **完整策划案**：可直接用于立项决策的 Markdown 文档
6. **评审报告**（可选）：质量诊断、优化建议、风险清单

## ⚡ 快速开始

在 Claude 中使用任一 Skill：

```
# 方式1：从主入口开始（推荐）
调用 topic-planning

# 方式2：直接使用某个Skill
调用 topic-research  # 市场调研
调用 topic-positioning  # 定位设计
调用 topic-review  # 质量检查
```

## 💡 使用建议

1. **第一次使用**：从 `topic-planning` 开始，系统会引导您
2. **核心环节**：`topic-positioning`（定位）最重要，值得花最多时间
3. **信息可信度**：`topic-research` 会严格验证信息，但您也应保持独立判断
4. **持续优化**：策划不是一次性的，可随时返回修改
5. **保存成果**：每个阶段的输出建议保存，便于后续参考

## 📌 注意事项

1. **联网搜索**：`topic-research` 需要联网搜索功能，请确保已启用
2. **信息时效**：市场数据有时效性，使用时请关注数据时间
3. **独立思考**：Skills 提供方法和框架，但核心决策需要您的专业判断
4. **迭代优化**：第一版策划案可能不完美，多次迭代会更好
