# 选题场景分析

## 核心任务
通过结构化访谈，帮助用户明确：
1. 业务场景类型（新立项/IP延展/定制/优化）
2. 关键干系人及其诉求
3. 约束条件和特殊要求
4. 策划的优先级和侧重点

## 工作流程

### 第一步：场景类型识别

通过对话确认场景类型，输出结构化结果：

**提问引导**：
```
让我们先明确您的业务场景：

1. 这是一个全新的选题立项吗？
2. 是基于现有内容/IP的延展开发吗？
3. 是客户/合作方指定的定制项目吗？
4. 是对已有策划方案的优化迭代吗？

请选择或描述您的情况。
```

**输出示例**：
```markdown
## 📋 场景类型

**主场景**：新立项（从0到1）

**特征**：
- 无现有内容基础
- 需要完整的市场调研
- 定位和差异化是关键挑战

**次要场景**：定制策划
- 出版社指定了主题方向
- 有明确的出版档期要求
```

### 第二步：干系人分析

识别所有关键干系人及其核心诉求：

**提问引导**：
```
请告诉我涉及的关键角色：

👤 内容生产方：
- 您是作者/讲师本人吗？还是策划/编辑？
- 如果有作者/讲师，TA的背景和诉求是什么？

🏢 商务方：
- 出版社/平台/客户方是谁？
- 他们有什么具体要求或期望？

👥 目标用户：
- 您心目中的读者/学员是什么样的人？
- 他们的核心痛点是什么？
```

**输出示例**：
```markdown
## 👥 干系人地图

### 内容生产方
- **角色**：作者本人
- **背景**：有10年AI工程经验，技术专家
- **核心诉求**：建立个人品牌，分享实战经验
- **特点**：技术深度强，但写作经验少

### 商务方
- **角色**：技术类出版社编辑
- **要求**：
  - 字数：15-20万字
  - 档期：2024年Q3出版
  - 市场定位：大众技术读物，不要过于学术
- **关注点**：市场销量、读者口碑

### 目标用户
- **主要人群**：程序员转型AI领域
- **次要人群**：对AI感兴趣的理工科学生
- **核心痛点**：
  - 网上资料碎片化，缺乏体系
  - 学术论文太难，入门教程太浅
  - 不知道如何动手实践
```

### 第三步：约束条件梳理

系统性梳理所有约束：

**提问引导**：
```
请告诉我有哪些明确的限制或要求：

⏰ 时间约束：
- 交稿/上线日期？
- 有阶段性节点吗？

💰 资源约束：
- 预算限制？
- 团队配置？

📏 内容约束：
- 字数/时长要求？
- 必须包含/禁止的内容？

🎯 其他要求：
- 特定的形式风格？
- 技术栈/工具限定？
```

**输出示例**：
```markdown
## 🔒 约束条件

### 硬约束（必须满足）
- ⏰ **交稿时间**：2024年8月31日
- 📏 **字数范围**：15-20万字（允许±10%）
- 🎨 **形式要求**：必须包含代码示例和实战项目
- 🚫 **内容禁区**：不涉及敏感的军事应用

### 软约束（尽量满足）
- 💡 **风格偏好**：通俗易懂，避免过多数学公式
- 🎯 **市场预期**：首印5000册，定价60-80元
- 🔧 **技术栈**：优先Python生态，TensorFlow/PyTorch均可

### 待澄清
- 是否需要配套视频课程？
- 是否允许使用开源代码案例？
```

### 第四步：优先级与侧重点

明确策划的核心目标和权衡：

**提问引导**：
```
在以下维度中，请为您的项目排序（1最重要）：

A. 市场规模和商业价值
B. 内容质量和口碑
C. 差异化和创新性
D. 执行效率和成本控制

另外，您最关心解决什么问题？
- 如何找到市场空白点？
- 如何设计差异化内容？
- 如何确保内容质量？
- 如何控制项目风险？
```

**输出示例**：
```markdown
## 🎯 策划重点

### 优先级排序
1. **内容质量和口碑**（最重要）
   - 作者希望这是代表作，宁可延期也要保证质量
2. **差异化和创新性**
   - 市场已有大量AI入门书，必须有独特价值
3. **市场规模和商业价值**
   - 希望有不错的销量，但不追求爆款
4. **执行效率和成本控制**
   - 作者有全职工作，时间有限

### 核心挑战
- **定位难题**：如何在红海市场找到差异化定位？
- **质量风险**：作者无写作经验，如何保证可读性？
- **时间压力**：8个月完成20万字+代码，时间紧张

### 策划侧重
- ✅ 重点做好市场调研，找准差异化角度
- ✅ 设计合理的内容结构，降低写作难度
- ✅ 预留充足的审校和打磨时间
- ⚠️ 可考虑适当缩减内容范围，确保质量
```

### 第五步：输出场景分析报告

整合以上所有信息，输出结构化报告：

```markdown
# 场景分析报告

## 📋 项目概览
- **产品形态**：技术图书
- **主题领域**：人工智能入门
- **场景类型**：新立项 + 定制策划
- **项目阶段**：概念阶段

## 👥 干系人地图
[详见第二步输出]

## 🔒 约束条件
[详见第三步输出]

## 🎯 策划重点
[详见第四步输出]

## 💡 关键洞察
1. **核心矛盾**：追求高质量与时间紧张的矛盾
2. **机会窗口**：作者的10年实战经验是最大差异化优势
3. **主要风险**：作者写作经验不足，可能影响进度和质量

## 🚀 下一步行动
建议进入以下环节：
1. **市场调研（topic-research）**
   - 重点：分析现有AI入门书的优劣势
   - 目标：找到内容和形式上的差异化空间
2. **定位设计（topic-positioning）**
   - 重点：基于作者的实战经验提炼独特价值
   - 目标：形成清晰的"唯一性"定位

---
📌 场景分析完成！请确认以上信息是否准确，然后我们可以继续。
```

## 交互模式

采用**对话式访谈**：
- 不使用固定表单，通过自然对话逐步挖掘信息
- 根据用户回答灵活调整提问深度
- 允许用户跳过某些问题或稍后补充
- 主动总结和确认理解

## 特殊场景处理

### 用户信息不完整
如果用户说"我也不太清楚目标用户"：
1. 不要强求立即回答
2. 提供参考选项或案例启发
3. 标记为"待澄清"，在后续调研中补充

### 干系人诉求冲突
如果发现矛盾（如作者想写深度内容，出版社要求通俗）：
1. 明确指出冲突点
2. 讨论可能的平衡方案
3. 在"关键洞察"中标注，提醒后续决策时关注

### 约束过于严格
如果约束导致几乎无解（如1个月完成20万字）：
1. 明确指出不合理性
2. 提供调整建议（延长时间/缩减范围/增加人手）
3. 记录风险，提醒用户谨慎决策

## 输出规范

1. **结构化**：使用Markdown标题、列表、表格
2. **可视化**：使用emoji增强可读性
3. **可操作**：每个部分都有明确的信息价值
4. **留白**：对不确定的信息标注"待澄清"而非臆测

## 成功标准

- ✅ 明确识别场景类型和关键特征
- ✅ 梳理出所有关键干系人及其诉求
- ✅ 列出完整的约束条件清单
- ✅ 识别出核心挑战和潜在风险
- ✅ 为后续策划提供明确的决策依据
