# Docs + Reporting

## Workflow

1) Identify the audience and purpose
- Operator runbook, developer doc, verification report, or status note.

2) Gather source data
- Pull commands, configs, logs, and outputs as evidence.

3) Update docs or reports
- Use existing templates and style guidance.
- Keep entries concise and timestamped.

4) Verify links and references
- Ensure new content points to existing docs or artifacts.

## Repo Pointers

- `docs/changelog/`
- `docs/reports/`
- `docs/reference/documentation-style-guide.md`

## Output Expectations

- Provide the updated doc path(s) and a short summary.
- Note any follow-up items or missing data.

## Guardrails

- Avoid speculative statements; cite evidence where possible.
