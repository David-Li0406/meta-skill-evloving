<action>

   - docs/prd/PRD.md
   - docs/prd/product-vision.md

2. Conduct a comprehensive analysis covering:
   - **Product Strategy**: Market fit, user adoption, competitive positioning

   - **Technical Feasibility**: Implementation challenges, scalability, performance
   - **Business Viability**: Success metrics, resource constraints, timeline reality
   - **Completeness**: TBD sections, missing information, unclear requirements
3. Identify 4-6 critical risks or gaps across these dimensions
4. Provide actionable recommendations for each issue
5. Create a separate analysis file: PRD_REDTEAM_COMMENTS.md

### Analysis Framework:

**Risk Categories to Evaluate:**

- **User Adoption Risks**: Will users actually behave as assumed?
- **Technical Implementation Risks**: Can this be built with stated constraints?
- **Business Model Risks**: Are success metrics realistic and measurable?
- **User Experience Risks**: Where might users get frustrated or confused?
- **Scalability Risks**: What happens if usage exceeds expectations?
- **Integration Risks**: How will this work with existing systems/processes?
- **Timeline/Resource Risks**: Are we being realistic about what can be delivered?

### Red Team Perspective:

- Challenge every assumption
- Look for what could go wrong
- Identify incomplete or unclear sections
- Question whether solutions actually solve stated problems
- Consider edge cases and failure scenarios
- Evaluate if success metrics can actually be measured

</action>
