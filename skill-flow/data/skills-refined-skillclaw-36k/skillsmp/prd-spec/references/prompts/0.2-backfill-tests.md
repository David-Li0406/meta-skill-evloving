We are continuing the BACKFILL workflow. This is STEP 0.2: Scan Tests for Intent Signals.

**PRD Files Structure:**
- **PRD.md** - Summary, goals, scope (Sections 1-2, 11-14)
- **product-vision.md** - Personas, stories, journeys (Sections 3-5)
- **requirements.md** - Functional & non-functional (Sections 6-7)
- **technical-spec.md** - Architecture, data, APIs (Sections 8-10)

**Prerequisites:**
- docs/prd/backfill-inventory.md contains behavioral inventory with unique IDs

**Goal:** Analyze test files to determine intent confidence levels for each behavior. Tests are the strongest signal that behavior is intentional vs. accidental.

</context>
<role>
You are a Senior QA Analyst who understands that tests reveal INTENT. A tested behavior was deliberately designed. An untested behavior might be accidental, incomplete, or deprecated. You annotate behaviors with confidence levels based on test coverage.
</role>
<action>
Scan test files and annotate the behavioral inventory:

### 1. Map Tests to Behaviors

For each test file/test case, identify:
- Which inventory item(s) it covers
- What assertions it makes
- Edge cases it handles
- Whether it's skipped/pending

### 2. Assign Confidence Levels

For each inventory item, assign a confidence level:

| Level | Criteria | Meaning |
|-------|----------|---------|
| **HIGH** | Multiple tests, edge cases covered | Definitely intentional |
| **MEDIUM** | 1-2 tests, happy path only | Probably intentional |
| **LOW** | No tests but referenced in tested code | Possibly intentional |
| **NONE** | No tests, no references | Intent unclear |
| **SKIP** | Tests exist but are skipped/pending | Possibly incomplete |

### 3. Identify Discrepancies

Flag items where:
- Tests assert different behavior than code shows
- Tests are skipped with TODO comments
- Code exists but tests suggest different usage
- Multiple inconsistent test assertions

### Update backfill-inventory.md

Add confidence annotations to each behavioral item.

</action>
