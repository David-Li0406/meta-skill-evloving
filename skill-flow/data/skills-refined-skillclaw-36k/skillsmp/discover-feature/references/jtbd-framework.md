# Jobs to be Done (JTBD) Framework Reference

## Core JTBD Statement Template

```
When [SITUATION/TRIGGER],
I want to [MOTIVATION/GOAL],
so I can [EXPECTED OUTCOME].
```

## Discovery Question Bank

### Situation/Trigger Questions

**Basic Trigger**
- "What situation or moment triggers the need for this feature?"
- "When do users realize they need this capability?"

**Context Questions**
- "Where is the user when this need arises? (physically, in-app, emotionally)"
- "What were they doing just before this need appeared?"
- "Is this need urgent or can it wait?"

**Frequency Questions**
- "How often does this situation occur?"
- "Is this a daily occurrence or occasional?"
- "Are there patterns to when this happens? (time of day, day of week, life events)"

### Motivation/Goal Questions

**Desired Action**
- "What does the user want to accomplish?"
- "If they had a magic wand, what would happen instantly?"

**Underlying Need**
- "Why do they want to do this? What's the deeper goal?"
- "What are they really trying to achieve beyond the immediate task?"

**Constraints**
- "What limitations do they face? (time, knowledge, context)"
- "What's preventing them from achieving this goal today?"

### Expected Outcome Questions

**Success Definition**
- "What does success look like for the user?"
- "How would they know the feature worked well?"

**Emotional Outcome**
- "How should the user feel after using this feature?"
- "What negative feeling should go away?"

**Measurable Outcome**
- "What changes after they complete this action?"
- "What becomes possible that wasn't before?"

## Current Alternatives Analysis

### Questions to Uncover Alternatives
- "How do users currently accomplish this? (even if poorly)"
- "What workarounds exist?"
- "What other apps or tools do they use for this?"
- "What do they do when this feature doesn't exist?"

### Alternative Categories
1. **Manual workarounds** - Paper, mental tracking, spreadsheets
2. **Competing products** - Other apps that solve this
3. **Adjacent features** - Existing features used incorrectly
4. **Avoidance** - User just doesn't do it
5. **Delegation** - Someone else handles it

## Pain Point Identification

### Pain Categories
1. **Time waste** - Takes too long
2. **Cognitive load** - Too hard to figure out
3. **Error prone** - Easy to make mistakes
4. **Context switching** - Requires leaving the app
5. **Missing information** - Can't make good decisions
6. **Friction** - Too many steps
7. **Anxiety** - Uncertainty about outcomes

### Pain Level Assessment
- **Critical**: Blocks the user completely
- **Major**: Significantly degrades experience
- **Minor**: Annoying but workable
- **Nice-to-have**: Would be better but current is okay

## JTBD Validation Checklist

After synthesizing the JTBD statement, validate:

- [ ] **Situation is specific** - Not vague, identifiable moment
- [ ] **Motivation is actionable** - Clear verb, achievable goal
- [ ] **Outcome is measurable** - Can determine if achieved
- [ ] **User confirms accuracy** - Resonates with their experience
- [ ] **Problem is worth solving** - Frequent enough, painful enough
- [ ] **Solution space is open** - Statement doesn't prescribe solution

## Example JTBD Statements

### Meal Logging
```
When I've just finished eating and I'm still at the table,
I want to quickly record what I ate,
so I can maintain accurate nutrition tracking without disrupting my day.
```

### Recipe Discovery
```
When I'm planning meals for the week and feeling uninspired,
I want to find recipes that fit my nutritional goals,
so I can eat healthy without spending hours on meal planning.
```

### Grocery Shopping
```
When I'm at the grocery store and see an unfamiliar ingredient,
I want to quickly check its nutritional value,
so I can make informed choices without derailing my diet.
```

### Progress Tracking
```
When I wake up and check my phone,
I want to see my nutrition progress at a glance,
so I can feel motivated and plan my day's eating accordingly.
```

## Interview Tips

1. **Ask "why" 5 times** - Get to root motivations
2. **Listen for emotions** - "Frustrated", "annoyed", "worried" reveal pain
3. **Watch for workarounds** - Creative solutions indicate strong need
4. **Note frequency** - Daily problems are more valuable to solve
5. **Identify switching triggers** - What would make them change behavior?

## Anti-Patterns to Avoid

- **Feature-first thinking**: "I want a barcode scanner" (solution, not job)
- **Vague situations**: "When I'm using the app" (not specific)
- **No outcome**: "I want to track food" (missing the 'so I can')
- **Multiple jobs**: Cramming too much into one statement
- **Assumptions**: Filling in gaps without user validation
