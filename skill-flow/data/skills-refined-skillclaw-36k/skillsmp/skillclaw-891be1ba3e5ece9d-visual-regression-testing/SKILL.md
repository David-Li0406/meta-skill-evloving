# Skill body

1. **Check Test Status**
   - Inquire about the current status of visual regression tests.
   - Request a summary of test results, including any failures or successes.

2. **Analyze Failures**
   - If there are failing tests, ask for details on the specific failures.
   - Review screenshot differences and visual bugs reported in the tests.

3. **Debugging Process**
   - Investigate the causes of visual changes by comparing expected vs. actual results.
   - Use tools provided by Vizzly to analyze diffs and identify issues.

4. **Modes of Operation**
   - This skill works in both local Test-Driven Development (TDD) environments and cloud modes, ensuring flexibility in usage.
