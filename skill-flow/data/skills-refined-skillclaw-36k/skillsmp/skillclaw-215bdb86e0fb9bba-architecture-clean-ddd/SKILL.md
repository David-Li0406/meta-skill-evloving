# Architecture Clean + DDD + Hexagonal

This skill provides guidelines and best practices for implementing Clean Architecture alongside Domain-Driven Design (DDD) and Hexagonal architecture. 

1. **Understand the Principles**: Familiarize yourself with the core concepts of Clean Architecture, DDD, and Hexagonal architecture.
2. **Define Boundaries**: Clearly define the boundaries of your application and its components.
3. **Layered Structure**: Organize your code into layers (e.g., presentation, application, domain, infrastructure) to promote separation of concerns.
4. **Dependency Rule**: Ensure that dependencies point inwards, with outer layers depending on inner layers.
5. **Use Interfaces**: Leverage interfaces to decouple components and facilitate testing.
6. **Focus on Domain**: Prioritize the domain model and business logic in your design.
7. **Iterate and Refine**: Continuously review and refine your architecture as the application evolves.

For detailed documentation, refer to the guidelines provided in the associated resources.
