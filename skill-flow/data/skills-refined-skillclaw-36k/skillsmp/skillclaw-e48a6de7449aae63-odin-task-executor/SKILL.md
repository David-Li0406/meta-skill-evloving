# Skill body

## Responsibilities

This skill is responsible for executing specific development tasks created by an ODIN plan and reviewing the quality of the completed tasks.

## Input Parameters

- **plan.yml path**: The path to the ODIN plan file generated by the ODIN planning process.
- **task ID**: The ID of the task to be executed or reviewed.

## Workflow

### 1. Read the Next Task

Use the following command to read the next task to be executed:

```bash
python3 scripts/read-odin-task.py <plan.yml>
```

The script returns task information including:
- `plan_dir`: Path to the plan directory
- `description_path`: Path to the description.md
- `task_id`: Task ID (e.g., 01-create-model)
- `task_detail_path`: Path to the task detail document
- `files`: Information on file changes (add/modify/delete)

**If there are no tasks to execute**:
- Output a message: "All tasks are completed."
- End the process.

### 2. Read Task Documentation

Read the task documentation to understand the overall goals and specific requirements:

```bash
Read <description_path>
Read <task_detail_path>
```

### 3. Execute Task Steps

Follow the steps outlined in the task detail document to execute the task.

**Execution Principles**:
- Adhere to the existing code style and architecture of the project.
- Follow the steps in the task document strictly.
- Confirm task completion based on validation criteria.

**Important Notes**:
- Only execute the current task; do not skip to other tasks.
- If requirements are unclear, ask the user for clarification.
- Ensure the code works correctly.

### 4. Update Task Status

After completing the task, update the status using the following command:

```bash
python3 scripts/update-odin-task.py <plan.yml> <task-id>
```

### 5. Return Execution Results

Return a summary of the execution results to the caller, including:
- Task ID
- Completed work details
- Any encountered issues (if applicable)
- Related file changes

### 6. Review Task Quality

After execution, review the task's code quality and compliance with standards.

#### Review Steps

1. **Read Task Documentation**: Understand the review standards by reading the task detail document.

2. **Check Task Completion**: Verify if the task meets the completion criteria based on the documentation.

3. **Audit Code Quality**: Conduct a thorough review of the code, checking for:
   - Code functionality and correctness
   - Naming conventions
   - Code style consistency
   - Error handling
   - Code consistency with project standards
   - Potential issues (performance, security, resource leaks)
   - Test coverage
   - Documentation completeness

4. **Generate Review Report**: Create a report based on the review findings, indicating whether the code passed or if issues were found.

5. **Handle Review Results**: If issues are found, ask the user if they want to fix them. If confirmed, proceed with the fixes and re-review.

## Task Status Definitions

- **finished: false**: Task is not completed.
- **finished: true**: Task is completed.

## Important Considerations

- **Sequential Execution**: Execute tasks in order from top to bottom.
- **Status Consistency**: Read the task before execution and update the status immediately after.
- **Validation of Completion**: Confirm task completion based on the validation criteria in the task document.
- **Timely Reporting**: Report progress and issues encountered during execution promptly.
