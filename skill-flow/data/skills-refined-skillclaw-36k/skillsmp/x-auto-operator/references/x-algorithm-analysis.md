# X (Twitter) 推荐算法深度分析

> 基于 X 开源算法仓库 (https://github.com/twitter/the-algorithm) 的深度解析
> 最后更新: 2026年1月

## 一、算法架构概述

### 1.1 整体流程

X 的推荐算法每天运行约 **50亿次**，每次处理耗时约 **1.5秒**，流程如下：

```
候选采集 (1,500条) → 排名评分 → 过滤筛选 → 内容混合 → 最终展示
```

### 1.2 核心组件

| 组件 | 功能 | 代码位置 |
|------|------|----------|
| **Home Mixer** | 主时间线构建服务 | `home-mixer/` |
| **Heavy Ranker** | 神经网络排名模型 | `projects/home/recap` |
| **SimClusters** | 社区发现和内容嵌入 | `src/scala/.../simclusters_v2` |
| **Real-Graph** | 用户互动关系预测 | `src/scala/.../interaction_graph` |
| **TwHIN** | 知识图谱密集嵌入 | `projects/twhin` |
| **TweepCred** | 用户信誉评分 (PageRank) | 内部服务 |

## 二、Heavy Ranker 评分公式

### 2.1 核心公式

```
Score = Σ (Weight_i × Probability_i)
```

Heavy Ranker 是一个 **MaskNet 并行架构** 的多任务学习模型，输出 10 个 0-1 之间的概率值。

### 2.2 官方互动权重 (2023年4月开源版本)

| 互动类型 | 权重 | 说明 |
|----------|------|------|
| 🔥 **作者回复你的评论** | **+75.0** | 最高权重！获得原作者回复极其重要 |
| 💬 **直接回复** | **+13.5** | 普通回复也很有价值 |
| 👤 **资料点击** | **+12.0** | 用户点击你的头像/资料 |
| 🗣️ **对话参与 v2** | **+10.0** | 进入对话并互动 |
| ⏱️ **停留时间 (2min+)** | **+10.0** | 用户在对话中停留超2分钟 |
| 🔄 **转发 (Retweet)** | **+1.0** | 基础转发 |
| ❤️ **点赞 (Like)** | **+0.5** | 最低正向权重 |
| 🎬 **视频观看 (50%+)** | **+0.005** | 看完视频一半以上 |
| ⚠️ **负面反馈** | **-74.0** | 用户选择"减少此类推文" |
| 🚫 **举报** | **-369.0** | 被举报严重降权 |

### 2.3 关键洞察

```
回复价值层级:
- 回复获得作者互动: 150× 点赞价值
- 直接回复: 27× 点赞价值
- 转发: 2× 点赞价值
- 点赞: 1× 基准值
```

## 三、内容类型权重

### 3.1 媒体类型加权

| 内容类型 | 权重加成 | 最佳实践 |
|----------|----------|----------|
| **原生视频** | **2-4×** | 2分20秒以内效果最佳 |
| **图片** | **1.5-2×** | 高质量信息图、代码截图 |
| **纯文字** | **1×** | 基准值 |
| **外部链接** | **0.5-0.7×** | 降权30-50%！ |
| **YouTube链接** | **0.3-0.5×** | 严重降权 |

### 3.2 外链降权机制

**算法依据**: X 希望用户停留在平台内，外链会导致用户流失。

**应对策略**:
```
✅ 正确做法:
   - 主帖发布核心内容
   - 外链放在第一条评论
   - 评论中引导"链接见评论"

❌ 错误做法:
   - 主帖直接放外链
   - 使用短链接服务
   - 多个外链堆砌
```

## 五、Real-Graph 互动关系

### 5.1 用户互动强度计算

Real-Graph 组件追踪用户间的互动历史，计算互动倾向分数：

**公开互动**:
- 点赞历史
- 转发历史
- 关注行为
- 回复频率

**私密互动**:
- 资料查看
- 推文点击
- 通讯录数据 (如有)

### 5.2 策略启示

```
与大V建立互动关系的路径:
1. 持续点赞其内容 → 被系统记录为"感兴趣"
2. 有价值的评论 → 引发大V回复
3. 大V回复你 → +75分 + 关系强度提升
4. 重复上述过程 → 你的内容更容易被推荐给该大V的粉丝
```

## 六、SimClusters 社区发现

### 6.1 算法原理

SimClusters 将 X 上的所有用户和内容映射到 **~145,000个社区向量** 中：

```
用户 → 关注图谱 → 社区向量 (InterestedIn)
推文 → 互动图谱 → 内容向量 (TweetEmbedding)

推荐匹配 = cosine_similarity(用户向量, 推文向量)
```

### 6.2 实战策略

```
如何被正确归类到 "AI/大模型" 社区:
1. 关注 AI 领域头部账号 (OpenAI, Anthropic, etc.)
2. 与 AI 内容持续互动
3. 发布 AI 相关内容并获得 AI 社区用户互动
4. 使用 AI 相关 hashtag
```

## 七、降权惩罚因素

### 7.1 内容层面

| 行为 | 惩罚 | 说明 |
|------|------|------|
| 发布攻击性内容 | **-80%** | 可见性大幅降低 |
| 全大写文字 | **降权** | 被识别为"喊叫" |
| 过多 hashtag | **降权** | 超过3个开始有负面影响 |
| 垃圾链接模式 | **严重降权** | 频繁发布相似外链 |
| 被用户屏蔽 | **累积降权** | 屏蔽越多，推荐越少 |

### 7.2 账号层面

| 因素 | 影响 | 说明 |
|------|------|------|
| 高关注/粉丝比 | **降权** | 关注远多于粉丝会被怀疑 |
| 短时间大量操作 | **限流** | 被识别为自动化行为 |
| 新账号无互动 | **冷启动困难** | 需要养号期 |
| TweepCred 低分 | **降权** | PageRank 算法计算的信誉分 |

### 7.3 TweepCred 信誉分

基于 PageRank 算法计算用户信誉：

```
影响因素:
- 粉丝质量 (被高质量用户关注更重要)
- 账号年龄
- 互动历史
- 内容质量
```

### 10.2 评分函数伪代码

```python
def calculate_tweet_score(tweet, user):
    # Heavy Ranker 预测各类互动概率
    probs = heavy_ranker.predict(tweet, user)

    # 加权求和
    score = (
        probs['replied_by_author'] * 75.0 +
        probs['reply'] * 13.5 +
        probs['profile_click'] * 12.0 +
        probs['conversation_engagement'] * 10.0 +
        probs['retweet'] * 1.0 +
        probs['like'] * 0.5 -
        probs['negative_feedback'] * 74.0 -
        probs['report'] * 369.0
    )

    # Blue Verified 加成
    if tweet.author.is_blue_verified:
        if user.follows(tweet.author):
            score *= 4.0  # 内网
        else:
            score *= 2.0  # 外网

    # 媒体类型加成
    if tweet.has_native_video:
        score *= 2.0  # 可高达4×
    elif tweet.has_image:
        score *= 1.5
    elif tweet.has_external_link:
        score *= 0.6  # 降权

    return score
```
