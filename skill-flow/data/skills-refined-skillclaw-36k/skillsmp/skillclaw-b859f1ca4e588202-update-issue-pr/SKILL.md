# Skill body

## Overview

This skill allows you to review and update the description of an issue or pull request (PR) in a systematic way, ensuring that all necessary information is clear and up-to-date.

## Workflow

### Step 1: Retrieve the Issue or PR

For an issue:
```bash
gh issue view <issue-number> --json body,title,author,assignees,comments
```

For a PR:
```bash
gh pr view <pr-number> --json number,title,body,url,isDraft
```

### Step 2: Review the Description

- **For Issues**: Check if the body contains `※ Generated by Claude`. If it does, determine if the analysis comment needs updating or if new comments are required.
- **For PRs**: Ensure the description clearly states:
  - Overview: What the PR aims to achieve and how.
  - Changes: Specific changes made.
  - Verification: Checklist items for confirming functionality.

### Step 3: Propose Updates

If updates are needed:
- Use `AskUserQuestion` to confirm proposed changes.
- For issues, suggest adding analysis comments or asking questions if information is lacking.
- For PRs, suggest modifications to the description and confirm with the user.

### Step 4: Implement Changes

If the user approves:
- For issues, update the body with the new analysis comments.
- For PRs, use:
```bash
gh pr edit <pr-number> --body "<new-body>"
```

### Step 5: Document Changes

Ensure that the changes made are documented in the respective sections of the issue or PR.

## Templates

### Issue Update Template
```markdown
## Overview

[Purpose and background of the task]

## Changes

- [ ] Task 1
- [ ] Task 2

## Notes
```

### PR Update Template
```markdown
This PR aims to achieve [what it does] and how it does it.

## Changes

- Change 1
- Change 2

## Verification

- [ ] Functionality works as expected
- [ ] No impact on existing features
- [ ] Build passes
```
